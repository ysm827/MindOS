# 📥 写入——往 MindOS 灌信息

## C1: 注入身份

**痛点**：每个 Agent 都不认识你，换个工具又得从零开始。

**流程**：

```
"读一下我的知识库，帮我把自我介绍写进 Profile"
  → 上传简历 / 粘贴自我介绍
  → Agent 自动结构化写入 Profile（技术栈、代码风格、沟通偏好）
  → 切换到 Claude Code："帮我搭个项目"
  → 自动用 pnpm + Next.js + Tailwind，无需解释
```

**效果**：换 3 个不同 Agent，都能正确读取你的偏好。

> 💬 `这是我的简历，读一下，把我的信息整理到 MindOS 里。`

---

## C2: 注入信息

**痛点**：文章、会议、网页——你读过的东西，Agent 不知道，下次要用还得重新找。

**流程**：

```
"帮我把这篇文章/会议纪要/网页存到 MindOS"
  → Agent 自动提取要点、关联已有知识、归档到对应目录
  → 文章 → Resources/  会议 → Projects/  竞品 → Products.csv
  → 下次讨论相关话题，Agent 主动引用
```

**效果**：注入后任意 Agent 搜索关键词都能找到，不再是孤立信息。

> 💬 `帮我把这篇文章的要点整理到 MindOS 里。`

---

# 🔄 使用——跨工具调用知识

## C3: 跨 Agent 无缝切换

**痛点**：同一个任务在不同工具间切换，每次都要重讲背景。

**流程**：

```
MindOS GUI → "帮我设计一个 TODO 应用的技术方案"
  → 方案自动写入知识库

Claude Code → "帮我按 MindOS 里的 todo-app 方案写代码"
  → 直接读取方案，按你的偏好搭建骨架

Cursor → "看看 todo-app 的进展，帮我优化组件结构"
  → 读取同一份方案 + 进度，继续干活
```

**效果**：3 个 Agent 之间零重复解释，进度自动同步。

> 💬 `帮我按 MindOS 里的 XXX 方案开始写代码。`

---

## C5: 手机记灵感，多 Agent 执行

**痛点**：灵感稍纵即逝，记了也散落各处，没人跟进执行。

**流程**：

```
手机 → "新功能想法：给知识库加健康度评分"
  → MindOS 自动归档、关联已有计划、拆解为 3 个子任务

回到电脑 →
  Claude Code → "按 knowledge-health 计划写技术方案"
  Cursor → "实现过期检测模块"
  Gemini CLI → "调研竞品的健康方案"
```

**效果**：一句灵感 → 结构化归档 → 多 Agent 无缝接力。

> 💬 `帮我把这个想法整理到 MindOS，拆解成可执行的子任务。`

---

## C6: 新项目冷启动

**痛点**：每次新项目都要重复说明技术栈、目录规范，首版经常不对。

**流程**：

```
"帮我启动一个新的 SaaS 项目"
  → Agent 读取你的 Profile（技术栈偏好）+ Startup SOP（初始化流程）
  → 一次性生成：pnpm + Next.js + Tailwind + ESLint + CI/CD + 双语 README
  → 首版即可跑通，零纠偏
```

**效果**：无 MindOS ~25min（含纠偏） → 有 MindOS ~4min（首版可用）。

> 💬 `帮我按 MindOS 里的 Startup SOP 启动一个新项目。`

---

# 📈 演进——知识越用越值钱

## C4: 经验沉淀为 SOP

**痛点**：踩坑经验关掉对话就丢了，下次还踩同样的坑。

**流程**：

```
踩坑 30 分钟，终于搞定 ESM mock 问题
  → "帮我把这次排查经验沉淀到 MindOS"
  → Agent 提炼为 SOP：问题描述 / 根因 / 解法 / 适用场景

下次遇到类似问题 →
  → Agent 搜索已有 SOP → 直接用正确方案，3 分钟搞定
```

**效果**：30min 踩坑 → 1 次沉淀 → 后续 3min 复用。

> 💬 `帮我把这次对话的经验沉淀到 MindOS，形成可复用的工作流。`

---

## C7: 主动调研 → 结构化入库

**痛点**：想了解几个竞品，得自己一个个查，查完结果散落在对话里。

**流程**：

```
"帮我调研 Obsidian、Notion、Mem 的 AI 能力"
  → Agent 主动搜索、整理 → 结构化写入 Products.csv
  → 生成对比矩阵 + 差异化分析报告
  → 永久入库，任何 Agent/项目都能查到
```

**效果**：让 Agent 替你跑腿调研，结果一次入库、全局复用。

> 💬 `帮我调研 X、Y、Z 这几个产品，结果写入 MindOS 产品库。`

---

## C8: 人脉管理

**痛点**：聊完一个人，承诺和跟进散落在对话里，容易遗忘。

**流程**：

```
"今天和张三聊了，他对 MindOS 感兴趣，愿意写评测，约了下周三再聊"
  → Agent 更新/新建 Connections/张三.md（交互记录、下一步、情绪）
  → 自动生成待办："周三与张三聊合作细节"
  → 下次提到张三，任何 Agent 都有完整上下文
```

**效果**：从聊天到记录到待办，一次闭环。

> 💬 `我今天和 XXX 聊了这些内容，帮我更新到 MindOS 并生成跟进待办。`

---

# 🔍 管控——人类保持最终控制

## C9: 审计与纠偏

**痛点**：Agent 记了什么你不知道，记错了也没法改，错误会持续放大。

**流程**：

```
发现 Agent 搭项目时用了 npm 而不是 pnpm
  → 打开 MindOS GUI 查看 Profile → 发现技术栈没提 pnpm
  → 直接编辑修正："包管理器：pnpm（禁止 npm/yarn）"
  → 所有 Agent 下次自动遵守
```

**效果**：人类发现 → 审查 → 修正 → 全局生效，完整闭环。

> 💬 `帮我检查 MindOS Profile 里的技术栈偏好是否正确，有错误帮我修正。`
