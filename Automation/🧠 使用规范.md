# 🧠 使用规范

本文件是 Automation 目录的总览与使用约定，Agent 执行任务前应先读取此文件。

## 💡 核心理念

把个人工作流程、偏好和配置沉淀到 Notion，让 Agent 在执行任务时直接读取这些 SOP 和偏好，做到**一次定义，随处执行**。

- 所有重复性操作 → 写成 SOP，Agent 按步骤执行，无需反复说明
- 所有个人偏好 → 集中记录在此，Agent 自动遵循，无需每次提醒
- 所有常用配置文件 → 存入 Files/，Agent 执行时直接引用

## 📁 目录结构

```
Automation/
├── 🧠 使用规范.md          # 本文件，总览与约定
├── 🖥️ Server Setup.md     # 服务器初始化 SOP
├── 💻 MacBook Setup.md    # MacBook 初始化 SOP
└── Files/
    ├── 🔑 SSH 配置.md      # SSH 配置文件
    ├── 🔑 SSH Config.md    # SSH 密钥与脚本
    └── 📦 Mac 常用软件.md  # Mac 常用软件清单
```

## 🏷️ 文件命名规范

- 统一使用中文命名
- 每个文件名以 emoji 开头，体现文件类型或内容
- 多个词之间用空格分隔
- SSH、Server、Claude Code、Docker、Git 等通识词汇保留英文原名

## 🔗 文件引用规范

- 文件之间可通过相对路径互相引用
- 当某文件的**内容或文件名发生变动**时，应同步更新所有引用该文件的地方
- 本文件（使用规范.md）维护目录结构，是引用关系的总索引，每次新增、删除、重命名文件时必须同步更新

## 📝 格式规范

- 多行命令 → bash code block（缩进在对应 bullet 下方）
- 单行命令 → 行内 `code` 格式
- 章节标题和关键条目适当添加 emoji

## ✍️ 内容风格

- 面向 Agent 执行，步骤清晰，不需要过多解释
- 人工也会查阅，格式需友好易读
- 内容精炼，不啰嗦
