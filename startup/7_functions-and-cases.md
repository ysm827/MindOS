| 功能 | 一句话 | 状态 |
|------|--------|------|

| **Mind Space** | 按思维维度组织知识分区（项目/领域/兴趣） | ✅ |
| **Instruction** | 所有 Agent 遵守的全局治理规则 | ✅ |

## 二、使用场景分类

### A. 个人知识管理（谁都能用）

| 场景 | 怎么用 | 涉及功能 |
|------|--------|---------|
| **项目记忆** | 每个项目一个 Space，架构决策、技术选型、会议记录都存这里。切 Agent 不用重新讲背景 | Space + MCP + Bootstrap |
| **读书/学习笔记** | 读完一本书或课程，把核心收获整理成 Markdown。Agent 以后回答问题时会引用你的笔记 | 知识库 + 搜索 + 反向链接 |
| **日记/周报** | 用 Echo Daily 每天记一条。月底回顾时，AI 帮你发现趋势和模式 | Echo + Daily |
| **个人 Wiki** | 把散落在各处的知识（技术栈、工具链、联系人、流程）集中管理 | 知识库 + Graph + Backlinks |
| **目标追踪** | 用 Echo Growth 定义季度目标，用 Todo 渲染器追踪进度 | Echo Growth + Todo |

### B. 开发者工作流（核心用户群）

| 场景 | 怎么用 | 涉及功能 |
|------|--------|---------|
| **编码规范统一** | 把代码风格、命名规范、错误处理标准写成 Instruction。Cursor 和 Claude Code 都遵守同一套标准 | Instruction + MCP + 多 Agent |
| **架构决策记录（ADR）** | 每个重要的技术决策记一条：为什么选 PostgreSQL 而不是 MongoDB？Agent 以后写代码时参考 | Space + MCP |
| **Bug 修复知识库** | 遇到的坑和解法存起来。下次 Agent 遇到类似问题，自动引用你的经验 | 知识库 + 搜索 |
| **代码审查标准** | 把 CR 标准写成 Skill，Agent 按你的标准审查代码 | Skills + MCP |
| **多 Agent 协作** | Claude Code 写后端，Cursor 写前端，两者读同一套 Instruction，API 接口定义一致 | MCP + Instruction + ACP |
| **项目 Onboarding** | 新项目的技术栈、架构图、规范全在 Space 里。新团队成员的 Agent 一连就能上手 | Space + Bootstrap |
| **自动化工作流** | 定义 YAML 工作流：代码提交→自动审查→生成 changelog→更新文档 | Workflow + MCP |

### C. 创始人 / 产品经理

| 场景 | 怎么用 | 涉及功能 |
|------|--------|---------|
| **产品战略沉淀** | 把战略文档、竞品分析、用户反馈存在知识库。和任何 AI 讨论产品时都有完整上下文 | Space + MCP + 一键导入 |
| **用户访谈记录** | 每次访谈记关键词，AI 自动整理成结构化笔记，归档到对应主题 | 一键导入 + AI Organize |
| **决策日志** | 为什么做 A 不做 B？三个月后回看决策逻辑，不后悔也不重复 | Echo Past You + Git 历史 |
| **会议→行动** | 开会时敲关键词，Agent 主动调研、生成报告（Aha Moment #2） | 知识库 + MCP + Agent 主动执行 |
| **竞品监控** | 把竞品信息存在专门 Space，Agent 帮你做对比分析 | Space + 搜索 |
| **融资准备** | 战略、BP、pitch deck、财务预测全在一个知识库。和 AI 准备 Q&A 时自动引用 | Space + MCP |

### D. 内容创作者

| 场景 | 怎么用 | 涉及功能 |
|------|--------|---------|
| **写作风格规范** | 把你的文风偏好写成规则：用短句、不用"赋能"、段落不超过 3 句。所有 AI 写文章时遵守 | Instruction + Skills |
| **素材库** | 收集的金句、案例、数据存在知识库。AI 写文章时自动引用 | 知识库 + 搜索 + MCP |
| **系列文章管理** | 用 CSV 看板视图管理文章状态：选题→草稿→审校→已发布 | CSV 看板视图 |
| **多平台发布** | 同一内容用 Workflow 自动适配不同平台格式（公众号/小红书/Twitter） | Workflow + Skills |

### E. 研究者 / 学术

| 场景 | 怎么用 | 涉及功能 |
|------|--------|---------|
| **文献管理** | 每篇论文一个 Markdown，核心观点+自己的评注。双链图谱展示论文间关系 | 知识库 + Graph + Backlinks |
| **研究日志** | 每天实验记录用 Echo Daily，累积后 AI 帮你发现研究趋势 | Echo Daily |
| **方法论沉淀** | 把研究方法写成 Skill：数据清洗流程、统计分析步骤、论文写作规范 | Skills |
| **导师-学生知识传递** | 导师的方法论 Space 共享给学生的 Agent，学生的 AI 按导师标准工作 | Space + MCP + Team（远期） |

### F. 团队协作（远期 P3）

| 场景 | 怎么用 | 涉及功能 |
|------|--------|---------|
| **团队规范治理** | 统一的 Instruction 确保所有人的 Agent 行为一致 | Instruction + Team |
| **新人 Onboarding** | 团队知识库 + 规范 + FAQ，新人的 Agent 一连就"认识"团队 | Space + Bootstrap + Team |
| **方法论共享** | 资深成员的 Skill 共享给团队，经验可传授 | Skills + Team |
| **Agent 治理** | 谁的 Agent 改了什么文件，一目了然 | Agent Inspector + 审计 |

## 二点五、Agent 主动性的三个层次

上面的场景大部分是"用户存 → Agent 读"。但 MindOS 真正的杀伤力不在于存储，在于 **Agent 基于你的知识库主动做事**。

从被动到主动，分三层：

### 层次 1：Agent 自动管理知识（已实现）

用户只管往里丢东西，Agent 帮你整理、归类、关联。

| 场景 | 用户做什么 | Agent 做什么 | 已有 |
|------|----------|------------|------|
| **拖拽导入** | 把一堆文件拖进来 | AI 自动识别内容类型，归档到正确的 Space，创建关联链接 | ✅ 一键导入 |
| **笔记自动归档** | 在 inbox 写一条随手记 | Agent 读取内容，判断归属，移动到对应文件并更新交叉引用 | ✅ AI Organize |
| **会议记录整理** | 开会时敲几个关键词 | Agent 读取上下文，扩展为结构化笔记，归档到项目 Space | ✅ |
| **重复内容合并** | 不同时间写了类似的笔记 | Agent 检测语义重复，提示合并或关联 | 🔬 P2 记忆代谢 |
| **过时内容标记** | 知识库积累了 500+ 文件 | Agent 检测长期未引用的文件，提示归档或删除 | 🔬 P2 |

**这一层的价值：你不用花时间整理，知识库自己保持有序。**

### 层次 2：Agent 基于知识直接交付（部分实现）

Agent 不只是读你的知识库——它基于你的上下文，直接产出你需要的东西。

| 场景 | 用户做什么 | Agent 做什么 | 已有 |
|------|----------|------------|------|
| **上下文驱动的代码** | 让 Agent 写代码 | Agent 先读 Instruction，按你的规范写。不用每次讲"用枚举不用字符串" | ✅ |
| **知识驱动的写作** | 让 Agent 写一篇文章 | Agent 读你的素材库+风格规范，产出符合你调性的内容 | ✅ |
| **会议→调研报告** | 开会敲了关键词 | Agent 读上下文，主动做调研，生成 3 份完整报告 | ✅ Aha Moment #2 |
| **规则驱动的审查** | 提交代码 | Agent 按你的 Skill 里的审查标准自动 review | ✅ Skills + Workflow |
| **项目周报生成** | 什么都不做 | Agent 读取本周的文件变更+决策日志，自动生成周报草稿 | 🔬 P2 Workflow |
| **竞品分析更新** | 什么都不做 | Agent 定期检查竞品动态，对比你之前的分析，生成更新报告 | 🔬 P2 |

**这一层的价值：Agent 不只记住你——它替你干活，而且干出来的活符合你的标准。**

### 层次 3：Agent 预测意图并主动执行（前沿）

Agent 不等你下指令。它观察你的行为模式，预判你下一步需要什么，提前做好。

| 场景 | Agent 观察到什么 | Agent 主动做什么 | 阶段 |
|------|----------------|----------------|------|
| **意图识别** | 你写了几个关键词 | 判断"这是调研需求"，主动去做，不等指令 | ✅ 已演示 |
| **上下文预加载** | 你打开了某个项目的文件 | 预判你接下来要编码，提前把架构文档和规范推给 Agent | 🔬 P2 |
| **规则缺口提示** | Agent 执行中遇到没有规则覆盖的场景 | 主动问你"这里没有规则，你想怎么处理？要存为规则吗？" | 🔬 P2 |
| **决策辅助** | 你在纠结 A 还是 B | 读取你之前的决策模式，提示"你过去在类似场景选了 A，原因是..." | 🔬 P3 |
| **认知盲区提醒** | 你的规则覆盖了代码但没覆盖安全 | 提示"你在安全维度零覆盖" | 🔬 P2 映 |
| **习惯挑战** | 你连续 20 个项目都用 REST | 提示"最近 3 个项目更适合 GraphQL。有意的，还是习惯？" | 🔬 P3 启 |
| **知识衰减预警** | 某条规则 3 个月没被引用 | 提示"这条规则可能过时了，要更新还是归档？" | 🔬 P2 |

**这一层的价值：AI 不只是你的工具——它是你的认知伙伴。它比你更了解你的全貌。**

### 层次 4：Agent 反哺认知（映鉴校启的产品化）

前三层 Agent 在帮你做事。第四层不同——**Agent 帮你认识你自己。**

| 场景 | Agent 做什么 | 人获得什么 | 对应反哺 | 阶段 |
|------|------------|----------|---------|------|
| **纠正分布统计** | "你 60% 的纠正关于代码风格——这是你的核心审美" | 看到自己的判断模式 | 鉴 | 🔬 P2 |
| **认知时间线** | "你 3 个月前说'用字符串'，现在说'用枚举'。你的标准在进化" | 看到自己在成长 | 映 | 🔬 P2 |
| **规则矛盾检测** | "规则 A 说'函数不超过 20 行'，规则 B 说'一个函数做完整一件事'。你信哪个？" | 被迫做更深层的判断 | 校 | 🔬 P2 |
| **盲区地图** | "你的规则覆盖了 5 个维度中的 3 个。测试和安全是空白" | 看到认知地图的空白 | 映 | 🔬 P2 |
| **碎片→原则提炼** | "你的 50 条规则中有 18 条在说同一件事。可以归纳为 1 个原则" | 从碎片中涌现方法论 | 启 | 🔬 P3 |
| **路径依赖质疑** | "你 20 个项目都用 REST。最近 3 个更适合 GraphQL。有意的？" | 被挑战，打破惯性 | 启 | 🔬 P3 |
| **知识库健康报告** | "冗余率 12%，矛盾 3 处，6 个月未引用文件 47 个" | 知识体系的体检报告 | 校 | 🔬 P2 |
| **决策模式识别** | "你在技术选型上倾向保守，在产品方向上倾向激进" | 认识自己的决策性格 | 鉴 | 🔬 P3 |

**这一层的价值：AI 比你更了解你的全貌——因为它有你所有规则的全集、所有纠正的历史、跨时间的比较能力。它把这些你看不到的东西，变成你能看到的。**

### 四层递进的关系

```
层次 1：Agent 管理知识     "你不用整理"        → 省时间
层次 2：Agent 直接交付     "你不用动手"        → 省精力
层次 3：Agent 预测意图     "你不用开口"        → 省思考
层次 4：Agent 反哺认知     "你看到自己"        → 变更强

前三层：AI 越来越懂你
第四层：你越来越懂你自己
四层叠加 = 人机共演化
```

**这也是五层积累模型的产品化：**

| 积累层次 | 对应 Agent 主动性 |
|---------|-----------------|
| 1 记住 | Agent 记住你的背景 |
| 2 结构化 | **层次 1** — Agent 自动管理知识 |
| 3 反思 | **层次 2** — Agent 基于知识直接交付 |
| 4 方法论 | **层次 3** — Agent 预测意图 |
| 5 代理 | **层次 3 终态** — Agent 代持琐碎判断 |
| — | **层次 4 贯穿 3-5** — Agent 反哺认知，人变强 |

| 飞轮步骤 | 主要场景 |
|---------|---------|
| ① 痛点入口 | 编码规范统一、项目记忆、多 Agent 协作 |
| ② 判断积累 | ADR、Bug 知识库、写作风格规范、决策日志 |
| ③ AI 变准 | 代码审查标准、会议→行动、竞品监控 |
| ④ 人变清晰 | Echo 反思、目标追踪、方法论沉淀 |

## 四、场景优先级（按验证紧迫度）

### 第一梯队：当前就能验证的场景

这些场景功能已完备，只需找到用户验证：

1. **编码规范统一**——开发者最痛的点。Instruction 写一次，多 Agent 全遵守
2. **项目记忆**——切工具不用重新讲背景。止痛药场景
3. **会议→行动**——敲关键词，Agent 主动调研。已有 Aha Moment 素材

### 第二梯队：需要少量功能补全

4. **决策日志 + 回顾**——Echo Past You + 时间线对比
5. **写作风格规范**——面向内容创作者的横向扩展
6. **自动化工作流**——Workflow + 代码提交联动

### 第三梯队：需要 P2-P3 功能

7. **方法论共享**——Team 功能
8. **经验自动编译**——纠正 3 次自动生成规则
9. **认知时间线**——Echo + 版本对比
