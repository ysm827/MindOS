# MindOS 竞品分析

*整理时间：2026-03-27*

---

## 核心论点：MindOS 不在笔记赛道

```
Obsidian / Notion 解决的问题：
  "我有很多知识，怎么组织和检索？"
  → 人写笔记 → 人找笔记 → 人读笔记

MindOS 解决的问题：
  "我有很多判断，怎么让所有 AI 都知道？"
  → 人写判断 → Agent 读判断 → Agent 执行 → 经验回流
```

**Obsidian 的读者是人。MindOS 的读者是人和 Agent。**

一句话：Obsidian 是你的第二大脑，MindOS 是你和 AI 的共享大脑。

## 逐个竞品深度分析

### vs Obsidian

**他们做得更好的：**
- 编辑器打磨 5 年，体验极佳
- 插件生态 1000+，社区活跃
- 品牌认知成熟，"第二大脑"品类已建立

**MindOS 做得更好的：**
- Agent 能直接读写（Obsidian 无法做到）
- 纠正自动沉淀为规则，所有 Agent 复用
- 跨 Agent 上下文共享是核心功能
- INSTRUCTION.md 让所有 Agent 遵守同一套规则

**竞争策略：不替代，共存。**
> "你用 Obsidian 管你自己的笔记，用 MindOS 管你和 AI 之间的共享上下文。"

**什么时候直接竞争：** 只有用户不愿维护两个知识库时。应对：P2 阶段支持从 Obsidian vault 一键导入。

---

### vs Notion

**他们做得更好的：**
- 团队协作成熟（实时多人编辑）
- 数据库视图强大（表格、看板、日历）
- 3000 万用户的品牌认知

**MindOS 做得更好的：**
- 本地优先，数据不上云
- Agent 原生读写（Notion API 存在但 Agent 不原生支持）
- 纯 Markdown，零锁定（Notion 导出丢格式）
- 判断沉淀和经验回流

**竞争策略：打本地优先和 Agent 原生这两张牌。**
> "Notion 把你的数据锁在云端，AI Agent 要读还得走 API。MindOS 数据在你本地，Agent 直接读。"

---

### vs MemOS

**他们做得更好的：**
- 学术论文支撑（技术可信度高）
- 图数据库 + 向量检索（技术架构更深）
- 7.8K GitHub stars，社区势能

**MindOS 做得更好的：**
- 有 GUI，终端用户可直接使用（MemOS 是 Python SDK）
- 数据是人可读的 Markdown（MemOS 是结构化数据）
- 判断沉淀为 Skill/SOP（MemOS 只做记忆存储检索）
- 面向终端用户产品体验（MemOS 面向开发者/研究者）

**生态位差异：** MemOS 是基础设施层，MindOS 是应用层。两者可能共存甚至互补——MindOS 后端 RAG 未来可接入 MemOS 作为记忆引擎。

**风险：** 如果 MemOS 向上做 GUI，两者会直接竞争。

---

### vs OpenAI Memory

**他们做得更好的：**
- 亿级用户基数
- 零配置，内置在 ChatGPT 里
- 背靠 OpenAI 生态

**MindOS 做得更好的：**
- 透明可控（OpenAI Memory 是黑箱，你不知道它记了什么）
- 跨生态（OpenAI Memory 只在 OpenAI 产品内有效）
- 本地存储（OpenAI Memory 在云端）
- 人可主动治理（不是 AI 自己决定记什么）

**竞争策略：不正面打。** OpenAI Memory 服务的是"只用一个 AI"的人，MindOS 服务的是"同时用 3+ AI"的人。用户群不重叠。

---

### vs CLAUDE.md / .cursorrules（手动管理方案）

**手动方案的优势：**
- 零成本，不需要装任何工具
- 简单直接，一个文件搞定

**MindOS 做得更好的：**
- 一处修改全局生效（手动方案每个工具各维护一份）
- 上下文自动传递（手动方案要手动 @ 或粘贴）
- 纠正自动沉淀（手动方案要自己维护规则文件）
- 新增工具零配置（手动方案要新建一份 rules 文件）

| 维度 | 手动管理 | MindOS |
|------|---------|--------|
| 维护成本 | 每个工具各一份，要保持同步 | 一处修改，全局生效 |
| 上下文传递 | 要手动 @ 或粘贴 | 自动读取，零操作 |
| 纠正沉淀 | 纠正只在当次对话有效 | 写回知识库，永久生效 |
| 新增工具 | 要新建一份 rules 文件 | 配一次 token 即可 |

**关键判断：** 如果用户只用 1-2 个 AI 工具，手动方案够用。3 个以上，MindOS 的效率差距会越来越大。

## 核心竞争优势总结

MindOS 有且只有三件事是竞品都做不到的：

| # | 优势 | 为什么竞品做不到 |
|---|------|----------------|
| 1 | **判断沉淀为 Skill/SOP** | Obsidian/Notion 不面向 Agent；MemOS 只做存储不做沉淀；OpenAI Memory 是黑箱 |
| 2 | **跨 Agent 人可治理** | 每个 Agent 厂商只管自己的记忆；没有人做"跨 Agent 的统一治理层" |
| 3 | **纠正→规则→复用闭环** | 这是 MindOS 独创的"经验回流"机制，目前无竞品实现 |
| 4 | **人机共演化** | 不只是 AI 变准——AI 执行中暴露的盲区反哺人，人的判断力也在提升。竞品只做单向（人→AI 或 AI→人），没有做双螺旋 |

**一句话差异化：** Cursor 们让你写得更快，MindOS 让你想得更深。它们在训练 AI，MindOS 在利用 AI 训练你。

## 竞争策略总结

| 阶段 | 策略 | 具体做法 |
|------|------|---------|
| **P1（当前）** | 不抢现有用户 | 目标："还没有知识库"或"知识库散落各处"的多 Agent 用户 |
| **P2** | 降低迁移成本 | 支持从 Obsidian vault 一键导入；支持多协议适配 |
| **P3** | 自然吸引迁移 | 编辑器和生态追上来后，Obsidian 用户会因为 Agent 能力主动迁移 |

**核心原则：不在编辑器层和 Notion/Obsidian 竞争，在 Agent 治理层建立独占位置。**
