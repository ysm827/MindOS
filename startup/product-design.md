## 一、双层楔子策略

> 原始设计文档由 /office-hours 于 2026-03-26 生成，经 CEO Review 演进。

### 核心问题

MindOS 最关键的产品问题不是"还能加什么功能"，而是"用户第一笔钱到底是为哪个明确价值付费"。

### 用户痛点

| 痛点 | 描述 | 证据来源 |
|------|------|---------|
| 上下文割裂 | 换 Agent 靠人工搬运上下文 | 创始人经验 + 内测反馈（N≈10） |
| 治理黑箱 | Agent 记了什么，用户无法审查修正 | 创始人经验，待验证 |
| 判断断流 | 纠正和判断关掉就丢，无法复用 | 创始人经验，待验证 |

### 双层策略

| 层级 | 价值 | 对应功能 |
|------|------|---------|
| **获客层** | 项目记忆工作台——再也不用反复讲背景 | MCP Server、CLI、模板 |
| **留存层** | 判断沉淀——纠正和偏好自动变成规则 | Skills、Instruction、经验回流 |

**一句话：先卖"不用重复讲背景"，再让用户发现"我的判断在复利"。**

### 真实竞争对手

不是单一产品，而是"多工具拼接"的现状工作流——在 Claude / Cursor / Gemini CLI / 文档 / 笔记之间来回切换。

### 约束

- 品牌叙事已成熟，不能为找楔子丢掉长期品类叙事
- 产品能力已覆盖 GUI/MCP/Skills/Spaces/CLI，比单点工具宽
- "本地优先记忆"赛道开始拥挤，只讲"记忆"会被归类
- 最缺的是观察证据，不是功能列表

## 二、CEO Review 扩展决策

> 由 /plan-ceo-review 于 2026-03-26 执行，SCOPE EXPANSION 模式。

### Premise Challenge 结论

1. 楔子和地基对齐（GUI/渲染器是地基）
2. 首次体验路径偏长（需理解 Space/Instruction/Skill）
3. 证据缺口是最大风险
4. 创始人选择全面推进
5. 核心定位确认为"认知复利"

### 范围决策

| # | 提议 | 工作量 | 决策 | 理由 |
|---|------|--------|------|------|
| 1 | 一键导入现有项目 | M | 接受 | 直接解决"路径偏长" |
| 2 | Agent 使用周报 | S | 延后 | 留存层，获客优先 |
| 3 | 经验沉淀按钮 | M | 接受 | 把认知复利变成产品动作 |
| 4 | 首次连接 wow moment | S | 接受 | 极小投入，大情感冲击 |
| 5 | 知识库健康度仪表盘 | M | 延后 | P2 量化复利效果时再做 |

### 产品路线图

| 阶段 | 时间 | 重点 | 触发条件 |
|------|------|------|---------|
| P1 | Month 1-6 | MCP + GUI + Skills + CLI + 桌面端 | — |
| P1.5 | Month 4-8 | 一键导入 + 沉淀按钮 + wow moment | 100+ 周活跃知识库 |
| P2 | Month 8-14 | RAG + Knowledge API + Pro 订阅 | MAU 500+ 且留存 >30% |
| P3 | Month 14-24 | Team + Agent Governance + 垂直应用 | Pro 用户 200+ |
| P4 | Year 3+ | Context API 开放平台 | ARR $200K+ |

### 核心概念

| 概念 | 定义 | 用户何时需要理解 |
|------|------|----------------|
| **Space** | 按思维方式组织的知识分区 | P1（文件夹，Agent 能读） |
| **Instruction** | 所有 Agent 遵守的治理规则 | P1.5（写一次，全遵守） |
| **Skill** | 沉淀的判断转化的可复用执行手册 | P2（纠正自动变规则） |

## 三、验证计划

### 融资前必须完成

1. 找 10 个非朋友用户深度访谈，量化"反复讲背景"的时间成本
2. 找 3 个用户愿意付 $8/月测试 Pro 功能
3. 跟踪 MCP 协议在 OpenAI/Google 生态的采纳情况

### 观察要点

只观察三件事：
1. 用户第一步去哪里找项目记忆
2. 在哪一步开始卡住
3. 有没有自发把判断或 SOP 写回系统

## 四、10x Vision

MindOS 成为 AI 时代的认知复利引擎：
1. 用户正常工作，判断和偏好持续沉淀
2. 任何新 Agent 接入，零配置读到完整认知资产
3. 每次使用 AI 都在积累——用的越久越离不开

**Platonic Ideal：**
> 你打开 MindOS，什么都不用做。它已经知道你在做什么项目、你的偏好是什么、你上次纠正了 AI 哪里。你切到 Cursor，Cursor 知道你的架构决策。切到 Claude Code，Claude 知道你的规范。你从来不需要"交代背景"——因为背景一直在那里，且由你审核。
>
> 用户的感受："我终于不是在伺候 AI，而是 AI 在伺候我积累下来的判断。"
