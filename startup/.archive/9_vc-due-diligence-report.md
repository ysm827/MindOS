# MindOS VC 尽调报告（模拟）

*模拟视角：AI/Developer Tools 赛道 VC 合伙人 | 2026-03-31*

---

## 总体评级：WATCHLIST

暂不投，持续关注。创始人 vision 清晰、品类思考有深度、产品已有形态。但缺乏任何需求验证证据，market size 存疑，单人团队风险高。建议 6 个月后 re-evaluate。

---

## 一、Deal Breakers

### 1. 零需求验证

BP 自己写了"痛点来自创始人经验和口头反馈，尚未验证"。在要 $500K 来验证一个还没证明存在的需求。

最低证据门槛：
- 5 个独立用户（非朋友）持续使用 2 周以上
- 至少 1 个用户主动说"如果你收费我愿意付"
- 用户行为数据：每周 MCP 读写次数、知识库文件增长率

**建议：先用 $0 验证需求，拿到证据后再融。**

### 2. "多 Agent 用户"规模存疑

SAM 280 万（2800 万开发者的 10%）。但"同时使用 3+ AI Agent"的人 2026 年真实数字可能只有 10-50 万。其中有多少觉得"重复交代背景"值得付费解决？可能 80% 觉得"粘贴一下 CLAUDE.md 也没那么烦"。

## 四、合伙人会议会问的 5 个问题

1. **"30 个内测用户里，有几个是每周都在用的？频率是什么？"**
   → 如果答案是"不确定"，deal 会被 pass。

2. **"如果 Claude Code 明天内置 Memory 共享，你的用户会留下还是走？"**
   → 需要具体用户证据，不是"我们有 Skill/SOP"的理论回答。

3. **"你最活跃的那个用户，他每天用 MindOS 做什么？流程是什么？"**
   → 如果不能描述真实用户的具体工作流，说明还不够了解用户。

4. **"为什么是你？你的 unfair advantage 是什么？"**
   → 独立开发者 + 全栈 + AI 理解是优势但不够独特。什么是只有你才有的？

5. **"6 个月后你拿什么数据来融下一轮？"**
   → $500K 的退出条件？MAU 只有 100 会 pivot 还是继续推？

## 六、给创始人的建议

### 别融资

现在融资时机不对。没有牵引力数据，估值会被压得很低。先用 $0 跑出 PMF 信号。

### 这周就开始用户访谈

访谈清单已写好（`7_user-interview-guide.md`）。别再写文档——去聊。

### 找一个 co-founder

不一定要技术——找一个懂增长或懂销售的人。一个人做产品可以，一个人做公司不行。

### 6 个月后带这些数据回来

| 数据 | 目标 |
|------|------|
| 周活跃知识库（MCP 读写 >0） | 100+ |
| 深度用户案例 | 5 个（工作流、频率、留存） |
| 付费意愿证据 | 至少 1 个用户 |
| Co-founder | 在身边 |

**有这些 → 种子轮 $1M，估值 $5-8M 完全合理。**
**没有这些 → $500K，估值 $2M 都难。**

## 七、天使轮视角修正

> 上述评级偏 Series A 标准。天使轮逻辑不同——天使买的是创始人洞察力、愿景规模和执行可信度，不是数据。

### 天使轮 Scorecard

| 维度 | 评分 | 理由 |
|------|------|------|
| 创始人洞察 | 9/10 | "AI 时代判断力是稀缺资源"洞察极锐利。四阶效应推导、认知复利品类定义有独创性。"人机共演化"理念（不只是 AI 变准，人也在变强）是更深一层的差异化。 |
| 愿景规模 | 8/10 | "AI 时代认知基础设施"是 $10B+ 机会。双杠杆（Vibe Coding + IP）提升天花板。 |
| 执行可信度 | 8/10 | v0.6.29 已交付 GUI + MCP + ACP/A2A + Workflow 编排 + 一键导入 + CLI + 桌面端 + Skills。不是 PPT，是工程事实。 |
| 市场时机 | 7/10 | MCP 标准化正在发生，"认知外化"位置空白。但 MCP 能否成为行业标准仍有不确定性，多 Agent 用户规模存疑。 |
| 验证意愿 | 5/10 | 有用户访谈清单但还没行动。startup/ 有 8 个战略文档——thinker 能力已证明，doer 信号需要加强。 |
| 团队 | 5/10 | 天使轮 1 人可以接受，但没有招 co-founder 的行动信号。能招到人本身是领导力的证明。 |
| **综合** | **7.0/10** | **可投但有条件** |

### 为什么是 7.0 而不是更高或更低

**不是 5.8（Series A 标准太严）：** 天使轮不要求数据和增长曲线。创始人洞察力和执行力在天使轮的权重远大于牵引力。

**不是 7.8（过度修正太宽松）：** 天使投资人确实更看愿景，但好的天使也会区分"还没验证"和"不打算验证"。战略文档的深度证明了思考能力，但也暴露了一个风险——创始人是不是更享受思考而非行动？

**7.0 的含义：** 值得天使投资，但投资人会附加条件。

### "可投但有条件"具体意味着什么

天使投资人可能会说：

> "我愿意投 $100K。但有两个条件：
> 
> 第一，拿到钱的前 30 天内完成 10 个用户访谈，给我看记录。如果 10 人里 7 个说'这不是我的问题'，我们坐下来聊 pivot。
> 
> 第二，3 个月内找到一个 co-founder 候选人——至少在聊。不要求明天找到，但要看到你在找。"

这种有条件的天使投资很常见。不是卡你，是帮你——这两个条件恰好是你最需要做的事。

### 修正建议：先天使轮，再 Seed

| 路径 | 金额 | 估值 | 需要什么 | 投资人看什么 |
|------|------|------|---------|------------|
| **天使轮（推荐）** | $100-200K | $1-2M | 愿景 + 产品 demo | 创始人洞察力 + 执行力 |
| Seed 轮（6 个月后） | $500K-$1M | $5-8M | + 100 WAU + 付费信号 | 数据 + 团队 |

天使轮拿的是"验证的权利"，不是"规模化的资本"。

### 最有说服力的天使 Pitch

**1. 洞察：** "AI 时代，执行归零后，判断力是唯一稀缺资源。但没有人在做'帮人系统性积累判断力'的工具。"

**2. 类比：** "Obsidian 是人的第二大脑，100 万用户证明人需要第二大脑。MindOS 是人和 AI 的共享大脑——现在 AI 也需要读这个大脑。"

**3. Demo：** 现场演示——写一条规则 → 连上 Claude Code → 自动遵守 → 切到 Cursor → 也遵守。30 秒让投资人亲眼看到"跨 Agent 共享判断"在发生。第三条最重要——天使靠直觉决策，好 demo 胜过 100 页 BP。
