### 核心术语

| 术语 | 定义 | 复利类比 |
|------|------|---------|
| **判断（Judgment）** | 用户对某个决策的选择与偏好——比如"API 错误一律用枚举而非字符串" | 复利的"本金" |
| **沉淀（Crystallize）** | 判断转化为可复用的 Skill（执行手册）或 Instruction（治理规则） | 复利的"利率" |
| **复用（Reuse）** | 所有 Agent 无需重复提问，直接读取已沉淀的判断并执行 | 复利的"收益" |

## 一、执行摘要

### 公司概述

MindOS 是 AI 时代的认知复利引擎。每个用户的判断、偏好和方法论在 MindOS 中持续沉淀，被所有 AI 工具复用。用的越久，沉淀越多，AI 越准，用户越离不开。

**品类定义：** 认知复利引擎（Cognitive Compound Engine）——不是笔记工具（Obsidian 做得更好），不是 AI 记忆系统（MemOS 做得更深），而是让你的判断持续积累、被所有 AI 复用、由你治理的认知资产平台。

- **阶段：** P1（已有用户，尚未付费）
- **产品状态：** v0.5.70，已发布至 npm，桌面端已支持 macOS/Windows/Linux
- **用户：** 内测群约 30 人，GitHub 开源
- **品牌主张：** 人类在此思考，Agents 依此执行。

### 核心问题

AI 工具只管输出，不管沉淀。你纠正了 AI 100 次"用枚举不用字符串"，但第 101 次它还是用字符串。你的判断没有被任何系统记住——不是效率问题，是尊严问题。

> **证据状态：** 以上痛点来自创始人自身使用经历和内测群口头反馈，尚未经过系统化用户观察验证。这是当前最大的已知缺口，详见第九章"风险与应对"。

### 解决方案

- **判断沉淀** — 你的纠正和偏好自动变成 Skill/SOP 规则，不再丢失
- **跨 Agent 复用** — 内置 MCP Server，所有 Agent 零配置直连，共享你的判断
- **人可治理** — Agent 读写留痕，你随时审查、修正、删除——你的认知资产，你说了算

### 市场机会（Bottom-Up 估算）

| 层级 | 计算方法 | 规模 |
|------|---------|------|
| **TAM** | 全球软件开发者约 2800 万 × 年均 AI 工具支出 ~$200 | ~$56 亿 |
| **SAM** | 其中 ~10% 为多 Agent 重度用户（~280 万）× 年均 $100 | ~$2.8 亿 |
| **SOM（Year 3）** | 280 万中触达 3.5%（~10 万注册）× 5% 付费 × $96/年 | ~$50 万 ARR |

> 注：SAM "多 Agent 用户上下文管理"是新品类，尚无第三方报告验证。以上为 bottom-up 假设。

### 融资需求

种子轮 $500K，用于：产品打磨（40%）、用户增长（30%）、团队扩展（20%）、运营（10%）。预期 runway 18-24 个月。

## 二、问题与市场机会

### 问题

| 痛点 | 描述 | 频率 | 证据来源 |
|------|------|------|---------|
| 上下文割裂 | 切换 Agent 靠人工搬运上下文，每次对话重新交代背景 | 每天多次（假设） | 创始人经验 + 内测群反馈（N≈10） |
| 治理黑箱 | Agent 记了什么、记对没有，用户无法审查和修正 | 每次使用（假设） | 创始人经验，待用户观察验证 |
| 判断断流 | 对话中的纠正和判断关掉就丢，无法沉淀复用 | 每次对话（假设） | 创始人经验，待用户观察验证 |

> **下一步验证计划：** 找 10 个非朋友用户做深度访谈，量化"反复讲背景"的时间成本（目标：证明平均每天 >15 分钟用于重复交代上下文）。

### 真实竞争对手

不是某个单一产品，而是用户当前"多工具拼接"的现状工作流：在 Claude / Cursor / Gemini CLI / 文档 / 笔记 / 本地文件夹之间来回切换，靠人手搬运上下文。

### Why Now

Vibe Coding 时代，"做出来"不再稀缺，"想清楚"才稀缺。AI 已经能写代码、做设计、跑流程，但所有 AI 工具都只负责输出，不负责沉淀。MCP 协议的标准化（2025-2026）使得跨 Agent 共享上下文第一次成为可能。

### 市场趋势

1. **多 Agent 工作流成为常态** — 开发者平均使用 3+ AI Agent
2. **MCP 协议被越来越多工具采纳** — 已支持：Claude Code、Cursor、Cline、Zed、Windsurf、VS Code（Copilot）等；尚未采纳：OpenAI 原生、Google Gemini 原生
3. **本地优先回潮** — 数据隐私意识增强，开发者偏好本地控制
4. **AI 记忆赛道升温** — OpenAI Memory、SuperLocalMemory 等产品出现，验证了市场需求

### 目标用户

**核心 Persona：** 同时使用 3+ Agent 的独立开发者/创始人——管理复杂 SOP、产品路线图和技术架构，需要跨 Agent 共享上下文。

**扩展用户：**
- AI-native 小团队（3-15 人）— 共享 SOP、Profile、项目记忆给团队 Agent
- 系统性思考者 — 重度 Markdown 用户（本地知识库 500+ 文件），追求本地控制与隐私

## 三、产品与解决方案

### 产品定义

MindOS 不是"给笔记加 AI"（Obsidian 的读者是人），也不是"给 Agent 加记忆"（MemOS 的用户是开发者）。MindOS 是**人和 AI 之间的治理层**——让你的判断持续沉淀、被所有 Agent 复用、由你审计和修正。

**Obsidian 是你的第二大脑，MindOS 是你和 AI 的共享大脑。**

### 产品策略：双层楔子

不是一上来就卖"个人成长平台"，而是分层推进：

| 层级 | 价值 | 对应功能 |
|------|------|---------|
| **获客层** | 项目记忆工作台——再也不用反复讲背景 | MCP Server、CLI onboard/token、模板系统 |
| **留存层** | 判断沉淀系统——纠正和偏好自动沉淀为 Skill | Skills、经验回流、Instruction 规则 |
| **差异化层** | 认知复利——人和 AI 一起成长 | 知识库演进、方法论沉淀、跨 Agent 复用 |

**一句话：先卖"再也不用反复讲背景"，再把用户带到"我的判断开始复利"。**

### 核心功能

**面向人类：**
- GUI 工作台：双模式编辑器（富文本 + Markdown）、AI 对话（⌘/）、全局搜索（⌘K）、Wiki Graph
- 11 个渲染器插件：TODO Board、CSV Views、Wiki Graph、Timeline、Agent Inspector 等

**面向 Agent：**
- MCP Server：stdio + HTTP 双传输，兼容 Claude Code、Cursor、Gemini CLI、Windsurf 等 10+ Agent
- MindOS Skills：结构感知路由，对话中的判断可沉淀为可复用 Skill
- 结构化模板：预设 Profile、Workflows、Projects 等目录结构

**基础设施：**
- CLI 工具链：onboard / start / sync / token，30 秒完成安装到 Agent 连通
- Git 自动同步：自动 commit/push/pull，冲突保留 `.sync-conflict`
- 安全：Bearer Token、路径沙箱、INSTRUCTION.md 写保护、原子写入
- 桌面端：macOS / Windows / Linux 原生应用

### 核心概念

| 概念 | 定义 | 对应术语 |
|------|------|---------|
| **Space** | 按用户思维方式组织的知识分区 | 结构层 |
| **Instruction** | 所有 Agent 都遵守的治理规则 | 控制层 |
| **Skill** | 沉淀的判断转化为可复用的执行手册 | 执行层 |

三者形成：**结构层（Space）→ 控制层（Instruction）→ 执行层（Skill）**

### 产品路线图

| 阶段 | 时间（预估） | 重点 | 状态 | 触发条件 |
|------|------------|------|------|---------|
| P1 | Month 1-6 | 核心体验：MCP + GUI + Skills + CLI + 桌面端 | 进行中 | — |
| P1.5 | Month 4-8 | 首次体验优化：一键导入、沉淀按钮、wow moment | 规划中 | 100+ 周活跃知识库 |
| P2 | Month 8-14 | RAG 集成、Knowledge API、Pro 订阅上线 | 规划中 | MAU 500+ 且 7 日留存 >30% |
| P3 | Month 14-24 | Team 层、Agent Governance、垂直应用 | 规划中 | Pro 付费用户 200+ |
| P4 | Year 3+ | Context API 开放平台、第三方生态 | 愿景 | ARR $200K+ |

### 不做什么

- 不做通用笔记工具 — 不在编辑器层和 Notion/Obsidian 竞争
- 不做自建富文本格式 — 依赖纯 Markdown/CSV，保证可迁移
- 不做实时多人协同编辑 — 支持团队异步共享
- 不做 Agent 训练/微调 — 只做知识存储和工具调用

| 判断沉淀与复用 | Skill/SOP | 无（专注记忆存储检索） | 无 | 无 | 无 | 无 |
| 面向用户 | 终端用户（开发者/创始人） | 开发者/研究者（Python SDK） | 终端用户 | 终端用户 | 终端用户 | 开发者 |

### MindOS 的差异化

1. **不只是记忆，而是可治理的上下文层** — Agent 读写留痕，人类可审计
2. **判断沉淀与复用** — 纠正沉淀为 Skill/SOP，竞品都没有这个能力
3. **人机共享心智** — 同一套知识同时服务人类阅读和 Agent 执行
4. **品类定义** — Personal Context OS，不是笔记工具，不是 Agent 记忆

### 重点竞品：MemOS

**MemOS**（[MemTensor/MemOS](https://github.com/MemTensor/MemOS)，7.8K stars，Apache 2.0）是目前最接近的竞品。它自称"AI Memory OS"，为 LLM 和 Agent 系统提供统一的记忆层。支持三种记忆类型（明文记忆、KV cache 激活记忆、参数记忆/LoRA），有学术论文支撑（2025.07），且已发布 Claude Code 插件。v2.0.10，2026 年 3 月仍在活跃迭代。

**MemOS vs MindOS 关键区别：**

| 维度 | MindOS | MemOS |
|------|--------|-------|
| 核心用户 | 终端用户（开发者/创始人） | 开发者/研究者（Python SDK） |
| 交互方式 | GUI 工作台 + CLI + MCP | Python API + REST API |
| 记忆形态 | Markdown 纯文本（人可读写） | 结构化数据（图数据库 Neo4j/向量） |
| 人可治理 | GUI 审计面板，人类直接编辑 | API 级别，需编程操作 |
| 判断沉淀 | Skill/SOP 经验回流 | 无（专注记忆存储和检索） |
| 学术背景 | 无 | 有论文（MemOS: A Memory OS for AI System） |
| 生态位 | 面向终端用户的"思考的地方" | 面向开发者的"记忆基础设施" |

**结论：** MemOS 是基础设施层竞品，MindOS 是应用层产品。两者可能共存甚至互补——MindOS 后端 RAG 未来可接入 MemOS 作为记忆引擎。但如果 MemOS 向上做 GUI，或 MindOS 不能在用户体验上拉开差距，两者会直接竞争。

### 最大威胁：Agent 厂商自建

**风险：** Anthropic/OpenAI 可能在 Agent 层原生实现跨 Agent 上下文共享。他们有协议、有用户、有数据。

**应对策略：**
- MindOS 的核心价值在"人可治理 + 判断沉淀"，不在"记忆存储"本身——厂商做记忆，MindOS 做治理层
- 保持跨生态兼容——不绑定单一 Agent 厂商
- 如果 MCP 未成为行业标准，Plan B 是支持多协议适配（MCP + OpenAI function calling + 自定义 API）

### 护城河

- **网络效应：** 知识库越丰富 → Agent 越好用 → 用户越不想走
- **数据沉淀：** 用户的判断、偏好、方法论沉淀在本地，迁移成本随时间增长
- **生态锁定：** Skill/模板生态建立后，社区贡献形成正向循环
- **标准先发：** MCP 协议的 Personal Context 层目前无标准实现

## 十、愿景

**让你的每次思考都在增值。**

### 认知复利飞轮

```
你写下判断 → 沉淀为 Skill/SOP → 所有 Agent 复用 → Agent 更准
    ↑                                                    ↓
    └──── 你纠正 Agent → 纠正被记住 → 下次不再犯 ────────┘
```

每转一圈，你的认知资产增加一点，Agent 输出质量提升一点。这就是认知复利。

### 18 个月目标

MindOS 成为多 Agent 重度用户的首选认知复利工具。5000+ 开发者依赖 MindOS 管理跨 Agent 上下文，Skill 社区贡献超过 100 个模板。

### 3 年愿景

MindOS 成为 AI 时代的认知复利引擎。不是用户主动"管理知识库"，而是：

1. 用户正常工作，判断和偏好持续沉淀
2. 任何新 Agent 接入，零配置就能读到完整的认知资产
3. 用户的感受是："我终于不是在伺候 AI，而是 AI 在伺候我积累下来的判断。"

### 长期使命

让每个人在 AI 时代都能积累属于自己的认知资产——判断不再随对话消散，而是持续增值、被所有工具复用。
