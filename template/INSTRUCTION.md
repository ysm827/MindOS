# Template Instruction Set

本文件定义 `template/` 目录下多语言预设模板的维护规则。

## 1. 目标

- 为用户提供可直接初始化的知识库预设。
- 支持中英文双语模板，并保持结构稳定。
- 降低 Agent 在不同语言模板间切换的认知成本。

## 2. 目录约定

`template/` 下应至少包含：

- `en/`: English preset
- `zh/`: 中文预设
- `README.md`: 模板入口说明
- `SETUP.md`: 初始化步骤
- `INSTRUCTION.md`: 本文件（维护协议）

## 3. 一致性规则

- `en/` 与 `zh/` 的目录结构必须保持同构。
- 同名功能目录在两种语言中都应存在，例如 `Profile/`、`Connections/`、`Workflows/`。
- 不同语言版本间应保持语义一致，不改变信息架构与执行意图。
- 在语义一致前提下，允许表达语气与文风不同。

## 4. 文案规则

- `en/` 使用英文文案；`zh/` 使用中文文案。
- 固定系统字段保持统一命名，便于 Agent 解析：
  - `README.md`
  - `INSTRUCTION.md`
  - `TODO.md`
  - `CHANGELOG.md`
- 结构性字段（例如记录模板字段名）可使用英文键名，确保跨语言可迁移。

## 5. 变更同步

发生以下变更时，必须同步更新 `en/` 与 `zh/`：

- 新增、删除、重命名一级目录
- 新增、删除、重命名公共模板文件
- 任何影响导航或检索路径的结构调整

建议同步（可分批完成）：

- 示例内容增强
- 描述性文案优化

## 6. 初始化建议

- 中文用户默认使用：`template/zh`
- 英文用户默认使用：`template/en`
- 初始化命令示例：

```bash
cp -r template/zh my-mind/
# or
cp -r template/en my-mind/
```

## 7. 安全边界

- 模板中不得写入真实密钥、Token、密码、个人隐私数据。
- 模板仅提供结构与示例，不承载敏感生产信息。

## 8. 维护原则

- 先保结构稳定，再做内容优化。
- 优先小步迭代，避免一次性大规模重组。
- 每次结构调整后，检查 `README.md` 与初始化命令是否仍准确。

## 9. 模板演进工作流（建议强制执行）

每次完善模板时，按以下顺序执行：

1. **定义变更意图**：明确这次是“结构变更”还是“文案变更”。
2. **先改信息架构**：先在一种语言里完成目录/文件骨架，再镜像到另一种语言。
3. **双语语义对齐**：保证 `en/` 与 `zh/` 语义一致，允许语气和措辞不同。
4. **同步入口文档**：更新 `template/README.md`、`template/SETUP.md` 以及受影响目录的 `README.md`。
5. **执行结构校验**：检查两套模板是否同构、是否缺少入口文件。
6. **最后再优化文案**：结构稳定后再做措辞润色，避免反复返工。

## 10. 变更类型与最小动作

### A. 结构变更（新增/删除/重命名目录、文件）

最小动作：

- 同步修改 `template/en/` 与 `template/zh/`
- 为新目录补齐 `README.md`（必要时加 `INSTRUCTION.md`）
- 更新相关上层 `README.md` 的目录树与用途说明
- 更新初始化相关说明（若路径或推荐入口变更）

### B. 文案变更（不改结构）

最小动作：

- 保持语义一致
- 可允许语气风格不同
- 不引入新的路径或字段名分叉

## 11. 完成定义（Definition of Done）

一次模板改动完成前，必须满足：

- `en/` 与 `zh/` 目录结构同构
- 新增目录均有 `README.md`
- 新增局部规则目录含 `INSTRUCTION.md`（如适用）
- `template/README.md` 与 `template/SETUP.md` 仍可直接指导初始化
- 没有悬空引用（被删除/重命名后未更新的路径）

## 12. 推荐校验命令

```bash
# 1) 对比 en/ 与 zh/ 目录结构（忽略文案，只看路径）
diff -u <(cd template/en && find . -print | sort) <(cd template/zh && find . -print | sort)

# 2) 检查每个一级目录是否有 README.md
for d in template/en/* template/zh/*; do [ -d "$d" ] && [ -f "$d/README.md" ] || echo "missing README: $d"; done

# 3) 搜索可能失效的旧路径引用（按需补充关键词）
rg -n "Profile/Connections|🔗 Network\.md" template README.md README_zh.md
```

## 13. 示例命名约定（`_example` / `_examples`）

凡是名称中包含 `_example` 或 `_examples` 的文件/目录，统一视为**示例模板**，不是用户自己的正式数据。

### 语义规则

- `_example`：单个示例文件或示例目录
- `_examples`：示例集合目录
- 这些内容用于演示结构、写法、字段，不作为长期真实记录

### 使用规则

- Agent 不应把 `_example` / `_examples` 内容当作用户事实来源
- 初始化后，用户应在正式目录中填写自己的内容，而不是直接把示例当生产数据
- 若需要保留示例，建议与正式文件并存，避免覆盖真实内容

### 维护规则

- 新增示例时，文件或目录名必须带 `_example` 或 `_examples`
- 若示例升级为正式模板，需移除该后缀并同步更新引用
- README 中引用示例时，应明确标注“示例用途”
