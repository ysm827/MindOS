# 任务后 Hooks — 提议格式

触发条件已列在 SKILL.md 中。仅在某个 hook 已确认触发后读此文件。

## 提议格式

一句话 + 具体目标路径/文件。用户同意后再展开。

| Hook | 提议模板 |
|------|---------|
| 经验沉淀 | 「要把这个经验记到 {相关经验文件} 吗？」— 格式：问题 → 原因 → 解决 → 规则 |
| 一致性同步 | 「{B} 引用了刚改的内容，要同步更新吗？」 |
| SOP 偏差 | 「执行过程与 {SOP 文件} 有偏差，要更新这个 SOP 吗？」— 动作：更新偏差步骤、追加踩坑、将 `<!-- last-used: -->` 设为今天 |
| 联动更新 | 「{关联文档} 的对应信息要同步吗？」 |
| 结构归类 | 「要归类到 {推荐目录} 吗？」 |
| 模式提炼 | 「这个操作重复了多次，要整理成 SOP 吗？」 |
| 对话复盘 | 「这段对话有些决策值得沉淀，要做个复盘吗？」 |

## 用户自定义 Hooks

在此处按同格式追加，例如：

```markdown
### 周报素材收集（优先级：中）
- 条件：更新了 Project 相关文件
- 提议：「要把这个变更记到本周周报素材吗？」
```
