## 1. 项目简介 (Project Overview)

### 1.1 解决什么问题

想象这样一个场景：一家 50 人的互联网公司，首席架构师离职了。他脑子里装着三年的系统演进判断——为什么这个模块要单独拆出来，为什么那条连线代表异步而不是同步，为什么这个边界要这样划。这些判断从来没有被任何工具记录下来。

三个月后，新来的架构师用 Lucidchart 从零开始画第一张图。他不知道团队的规范，不知道前任的决策逻辑，不知道哪些地方踩过坑。他需要至少六个月，才能画出一张团队认可的图。

**这不是工具问题，这是组织智能的流失问题。EvoDiagram 要解决的，是这个。**

现有工具和方法在将组织的隐性判断力转化为可传递的资产时，面临三重根本性断裂：

- **组织智能与演进断裂（核心）：** 绝大多数 AI 生成结果是静态产物，无法学习团队偏好、沉淀组织经验，也无法随项目迭代而自动演进。每一位离职的资深成员，都带走了无法复原的设计判断力。
- **领域经验壁垒断裂：** 专业图表设计蕴含深厚行业知识和隐性视觉规范，学习成本高昂。新成员难以产出符合团队标准的图表，阻碍协作效率与知识传承。
- **创作与智能体验断裂：** 传统工具依赖手动操作；现有 AI 工具（如 Mermaid）仅为"单次映射"，缺乏语义理解和美学考量，生成结果难以精细化微调。

EvoDiagram 旨在彻底弥合这三重断裂。它不仅仅是一个绘图工具，更是一个 AI 驱动的**组织设计智能系统**——通过多 Agent 协作机制与分层知识演化体系，将团队最优秀成员的隐性判断力，沉淀为可继承、可传递的组织资产，让每一个新人从第一天起就站在巨人的肩膀上。

### 1.2 三步从想法到图表

| 步骤 | 用户动作 | 系统行为 |
|:---:|---|---|
| **① 描述** | 输入自然语言、粘贴代码片段，或上传文档 | 结构提取 Agent 解析实体、关系与意图 |
| **② 生成** | 等待数十秒（可实时观看思考流） | 5 Agent 协同链完成风格设计 → 布局计算 → 初稿渲染 → 自适应优化 |
| **③ 精修与导出** | 在无限画布上拖拽调整，或用自然语言局部修改 | 仅增量重绘选中区域，其余结构保持稳定；导出为矢量/图片/代码 |

整个流程对新用户零学习成本，首次体验即可获得完整价值。

## 2. 业务价值说明 (Business Value)

EvoDiagram 的业务价值围绕一条清晰的增长飞轮展开：**极致提效获客 → 透明共创留存 → 组织智能沉淀付费**。三个支柱依次回答投资人最关心的三个问题——用户为什么来、为什么留、为什么付费。

**核心商业命题先行：** 个人用户因"画图快"进来，但真正的商业价值发生在团队层面——用户的每次编辑都在训练系统，每一位新成员的入职都在验证这份沉淀的价值。这是 EvoDiagram 区别于所有现有 AI 绘图工具的根本所在。

### 2.1 沉淀设计基因：组织智能驱动付费与扩张

> 核心命题：解决"组织智能无法传承"的根本问题——这是付费、扩张与护城河的来源。

**新人 onboarding 场景（核心价值锚点）：**

> 新工程师入职第一周，打开 EvoDiagram，系统已经加载了团队过去 18 个月的设计基因——配色规范、模块命名习惯、布局偏好、行业约束。他画出的第一张图，自动符合团队所有的隐性规范。他从来没有被培训过。
>
> 这个场景每次有新人入职就发生一次。

这是 EvoDiagram 的核心价值，也是企业续费的真实理由。EvoDiagram 通过三层机制将一次性图表升级为持续增值的组织资产：

- **团队经验蒸馏为组织智能：** 分层设计知识演化机制将顶尖架构师、资深设计师的隐性经验，通过系统学习与反思，转化为组织显性的"设计基因"。这些知识以可复用的约束和优化策略沉淀下来，新成员入职即可继承组织的设计智能——从"个人技能"到"组织智能"的跨越，不再依赖人员传帮带。
- **企业 VI 锁定为品牌主权：** 通过"品牌记忆注入"功能，企业可将视觉识别系统（品牌色盘、专用字体、Logo、排版规范等）转化为 AI 的硬性约束。所有团队成员产出的图表自动符合企业 VI 规范，无需人工审核——同一张图可在极简、手绘、企业 VI 间瞬时迁移，但品牌一致性始终锁定。
- **个人偏好沉淀为设计记忆：** 系统实时记录用户在画布上的手动微调行为（拖拽节点、调整连线、修改样式），通过"设计记忆"模块进行在线学习。Agent 逐渐理解并适应用户的个性化绘图习惯与审美偏好，在后续生成中主动迎合——用得越多，出图越"懂你"。

**付费逻辑：** 企业用户为"组织设计基因 + 品牌主权锁定"付费（Team/Enterprise 订阅），按**席位**而非生成次数计费——因为新人越多，设计基因的传递价值越大，定价单位与价值主张完全对齐。个人用户为"越用越聪明"的偏好记忆付费（Pro 订阅）。关键商业洞察：用户沉淀的设计记忆和企业 VI 约束构成强迁移成本——数据和偏好无法带走，形成天然的 lock-in 效应，支撑高续费率和向上销售（upsell）。

**扩张逻辑：** 单个用户引入 → 团队协同需求 → 企业级组织智能管控需求，形成自下而上的渗透路径（Bottom-up PLG）。每多一个团队成员使用，组织的设计基因库就更丰富，切换成本就更高。

## 3. AI 创新性说明 (AI Innovation)

> 核心命题：解决"画图难、画图慢"的即时痛点——这是获客引擎。

专业图表的创作长期受困于"高门槛、低效率"的双重瓶颈。EvoDiagram 通过多 Agent 协同与内置行业知识，将这一过程从小时级压缩到秒级：

- **时间成本锐减：** 无论是几百个节点的微服务架构图，还是复杂的科研流程图，用户只需输入自然语言描述，即可在数十秒内获得完整的专业图表。传统流程中耗费在手动布局、连线对齐、样式调整上的时间被彻底消除。
- **设计经验平权：** 内置的分层设计知识（行业规范、美学原则、布局策略）将资深设计师的隐性经验"民主化"。非设计背景的开发者、架构师和研究员，无需掌握复杂绘图技巧，仅通过自然语言即可产出符合行业标准的"出版级"图表。
- **三步完成创作：** 描述想法 → AI 生成初稿 → 画布上精修导出。整个流程对新用户零学习成本，首次体验即可获得完整价值，形成强获客转化。

**获客逻辑：** 用户因"一句话出图"的即时满足感而来。在 Canvas Bench 评测中，EvoDiagram 在内容完整性、视觉呈现和认知效用三个维度上均显著超越传统 Prompting 方案，确保首次体验的质量足够驱动口碑传播。

### 2.3 透明、可控、懂你：深度人机共创驱动留存

> 核心命题：解决"AI 不可控、改不动"的信任问题——这是留存壁垒。

"一次性生成"是现有 AI 绘图工具的通病：输出是黑盒，不满意只能推倒重来。EvoDiagram 通过透明化的人机共创机制，让用户始终掌握主导权，从而建立长期使用粘性：

- **思考流实时可视化：** 系统在生成过程中实时展示 5 个 Agent 的工作进度与中间决策（"思考流"），用户可以清晰看到 AI 在做什么、为什么这样做。这种技术透明化设计消除了等待焦虑，更重要的是建立了用户对 AI 决策的信任——不再是"赌一次生成结果"，而是"看着它一步步画出来"。
- **局部编辑与增量重绘：** 用户可以点击画布上的任意节点，仅对该局部发出自然语言指令（如"拆分此步骤，改为规划器与评审器并行"），系统只重绘选中区域，其余结构保持稳定。这彻底解决了"改一处、崩全图"的痛点，让 AI 真正沿着用户的思路演进，而不是每次都推倒重来。
- **关键节点介入与引导：** 用户可以在生成的任何关键节点暂停并修正 Agent 的决策方向，实现对创作过程的精细控制。AI 不再是不可控的黑盒，而是一个"听得懂指挥"的协作伙伴。

**留存逻辑：** 用户因"改得动、改得准"而留下来。局部编辑能力让用户的每次交互都在改善当前图表，而非从零开始，单次会话深度和回访率显著提升。竞品的黑盒生成模式无法提供这种交互深度，形成差异化留存壁垒。

EvoDiagram 的核心竞争力在于其前瞻性的AI架构与机制，实现了从"AI辅助绘图"到"AI驱动协同演进"的范式转变，尤其在以下三方面展现出卓越的创新性：

### 3.1 Spec-to-Instance：多 Agent 协同进化范式

EvoDiagram 摒弃了传统"单一Prompt渲染"的局限性，创新性地构建了一个由 5 个高度专业化 Agent 组成的协同进化链条，每个 Agent 遵循"策略(Specification) → 实例(Instance)"的生成逻辑，确保图表从语义到视觉的精准转化与优化：

| Agent | 阶段 | 核心功能与创新点 |
|---|---|---|
| **结构提取 Agent** | ① 提取 | 深度解析非结构化文本（代码、需求文档、会议纪要等），智能识别实体、关系及层级结构，构建精确的逻辑拓扑图。创新性地解决了复杂语境下"identity 碎片化"问题，确保语义理解的完整性与准确性，为后续 Agent 提供高质量输入。 |
| **风格设计 Agent** | ② 设计 | 基于语义重要性、用户偏好及企业 VI 规范，智能分配视觉权重，自动匹配配色方案、字体与形状规范。一键切换从手绘风到企业 VI 标准，保持表达与品牌统一。 |
| **布局计算 Agent** | ③ 计算 | 引入物理引擎模拟与 Bottom-up Sizing 几何优化算法，在生成坐标前精确预计算文本与图形的物理足迹。像细胞组织一样自动寻找最优排版位置与结构密度，从根本上解决节点重叠、连线交叉等传统 AI 绘图顽疾，实现"零冲突"布局。 |
| **渲染优化 Agent** | ④ 优化 | 将结构、样式、布局的优化结果精准转化为 TLDraw/Excalidraw 兼容的矢量画布数据，并围绕层级、标签与连线持续微调。具备"视觉诊断"能力，能模拟人类设计师的审美与逻辑判断，自动发现并修正局部冲突、连线混乱或布局失衡，并可发起多 Agent 修正循环，实现图表的自我优化与演进。 |
| **设计治理 Agent** | ⑤ 治理 | 把设计规则沉淀为可复用约束，实现组织级"万图如一"。将顶尖设计师的隐性经验转化为可编程的设计策略，确保团队产出在风格、规范和质量上高度一致，是驱动"设计基因沉淀"（见 2.3）的核心机制。 |

### 3.2 分层设计知识演化机制 (Knowledge Evolution)

EvoDiagram 借鉴人类学习与成长模式，构建了独特的三层记忆架构，实现了设计知识的持续演化与沉淀，有效打破了领域经验壁垒：

- **样本策略层 (Trajectory Memory):** 记录每一次用户交互、Agent决策及图表生成过程中的成功与失败案例。通过强化学习与反思机制，系统能够从中学习并优化其策略，实现"越用越聪明"的自适应进化，累积个性化设计经验。
- **领域指南层 (Domain Guide):** 预置并持续更新金融、互联网、生物医药、工程等垂直行业的绘图规范与最佳实践（例如，金融时序图需严格横向对齐，微服务架构图需采用特定图标）。这些指南作为可编程的约束，确保生成图表符合行业专业标准，实现领域经验的平权化。
- **通用原则层 (Universal Principles):** 沉淀跨领域的设计美学与认知心理学原则，如"负空间平衡"、"功能模块化"、"信息层级清晰"等。这些原则作为底层约束，指导Agent生成具有普适美感与高认知效用的图表，提升整体设计质量。

### 3.3 交互式偏好学习与意图驱动的自适应演进 (Agentic Learning & Adaptive Evolution)

系统不仅是被动执行指令，更能通过与用户的深度交互进行在线学习，实现高度个性化的适配与图表的自适应演进：

- **品牌记忆注入：** 允许用户上传特定组织或项目的视觉规范（如品牌色盘、专用字体、Logo等）。AI将其转化为领域指南层的硬性约束，确保生成结果自动符合企业视觉识别系统（VI），维护品牌一致性，降低人工校对成本。
- **交互式偏好学习：** 实时记录用户在画布上的手动微调行为（如拖拽节点、调整连线、修改样式），并通过"设计记忆"模块进行在线学习。Agent能够理解并适应用户的个性化绘图习惯与审美偏好，在后续生成中主动迎合，实现真正的"人机共创"与图表的个性化自适应演进。
- **意图驱动的风格迁移与快速迭代：** 用户可以通过模糊的自然语言指令（如"让它看起来更专业"、"改成手绘风格"、"突出核心路径"）驱动 Style Agent 调取关联的个性化记忆碎片或预设风格模板，实现一键式的图表风格重塑。这极大地提升了设计的灵活性与迭代效率，确保图表始终符合用户最新的需求与偏好，加速从概念到最终呈现的转化。

## 4. 技术实现说明 (Technical Implementation)

### 4.1 整体架构

EvoDiagram 采用模块化、可扩展的混合AI架构，结合了前沿的LLM技术与传统图形学算法，确保了系统的高效、稳定与智能：

- **智能调度层：** 基于 LangGraph 构建复杂的状态机，负责管理多Agent之间的上下文传递、任务分配、自省循环（Self-Reflection）与冲突解决。这一层是实现协同进化范式的核心，确保复杂逻辑的端到端处理，并具备强大的容错与自优化能力。
- **前端交互层：** 采用 React + Tailwind CSS 构建现代化用户界面，集成高性能 Canvas 引擎 (如 TLDraw/Excalidraw)，支持对象级的拖拽、实时编辑与多用户协同。我们通过优化渲染管线，确保在处理包含数百个节点的复杂图表时，依然能提供流畅、直观的用户体验。
- **布局算法增强：** 结合大型语言模型（LLM）的语义理解能力与传统的 Bottom-up Sizing 几何算法，在图表生成坐标前精确预计算文本与图形的物理足迹。这种混合方法从根本上避免了视觉冲突，保证了图表的物理准确性与美观度，是实现"零冲突"布局的关键技术。
- **知识库与记忆模块：** 采用 向量数据库（Vector Database） 存储分层设计知识（Trajectory, Domain Guide, Universal Principles），实现高效的知识检索与更新。这一模块是系统持续学习与演进的基础，通过语义搜索快速匹配最佳设计策略。

### 4.2 评价与训练体系：Canvas Bench

![Canvas Bench 数据集构建流程与概览](images/image2.png)

Canvas Bench 不仅仅是 EvoDiagram 的内部质量评测工具——它是这个领域迄今为止最系统化的图表质量量化标准。**谁制定标准，谁就拥有话语权。** 这是 EvoDiagram 成为行业基础设施的路径之一。

该评测集包含 2260 个真实世界样本，涵盖多种图表类型与复杂度，从以下三个核心维度对 AI 生成图表进行量化评估与持续优化：

- **内容完整性 (Content Integrity)：** 评估图表是否准确、完整地表达了输入文本中的所有逻辑节点与关系，无遗漏、无冗余，确保信息传达的准确性。（量化指标：F1 Score of Entity and Relation Extraction）
- **视觉呈现 (Visual Presentation)：** 评估图表的布局平衡性、美观度、可读性，以及是否存在节点重叠、连线混乱等视觉缺陷，旨在提供最佳的视觉体验。（量化指标：Overlap Ratio, Aesthetic Score based on expert evaluation）
- **认知效用 (Cognitive Utility)：** 通过用户测试与眼动追踪等方式，量化用户理解该图表所需的时间成本与认知负荷，旨在最大化图表的沟通效率与易用性。（量化指标：Time-to-Understand, Cognitive Load Index）

我们采用混合经验搜索（结合深度优先的垂直优化与广度优先的需求自适应）从用户交互轨迹中自动蒸馏设计知识，持续提升模型性能，确保 EvoDiagram 始终处于技术前沿。

## 5. 交互与设计说明 (Interaction & Design)

### 5.1 专业三段式布局与人机协同工作流

EvoDiagram 采用直观且高效的"三段式"界面布局，旨在实现AI生成与人工精修的无缝衔接，构建深度人机协同的工作流：

- **左侧：语义控制区。** 用户在此输入自然语言指令，并可实时观察到多Agent的"思考流"（Thought Stream）与中间决策过程，增强AI透明度与用户信任，使用户能更好地理解AI的决策逻辑。（核心交互：Prompt输入、Agent状态可视化）
- **中间：沉浸式画布。** 这是核心工作区，支持AI生成结果的直接展示与对象级交互。用户可以对AI生成的图表进行拖拽、修改、调整，实现"人机协同"的深度融合，确保最终产物完全符合用户意图。（核心交互：矢量对象编辑、实时协同）
- **右侧：属性与记忆调节区。** 用户可在此精细化调整AI的"设计偏好"参数，管理和上传品牌视觉规范，或查看历史设计记忆，实现对AI行为的个性化定制与引导，让AI真正成为用户的智能助手。（核心交互：参数微调、VI/偏好管理）

### 5.2 局部编辑与增量重绘

这是 EvoDiagram 与"黑盒一次性生成"类竞品最直观的差异化交互：

**交互流程：**
1. 用户在画布上**点击选中**一个或一组节点
2. 在左侧语义控制区输入**局部编辑指令**（如"拆分此步骤，改为规划器与评审器并行"）
3. 系统**仅增量重绘选中区域**，其余结构、连线和样式保持稳定
4. 用户对结果满意则确认，不满意可撤销或继续微调

**设计原则：**

- **最小化破坏半径：** 每次编辑仅影响选中子图。即使 AI 需要调整相邻连线或重新计算局部布局，也不会"推倒重来"整张图。这一机制从根本上解决了用户对 AI 修改的信任问题——"改一处，崩全图"的恐惧被消除。
- **保留用户投入：** 用户手动调整过的节点位置、自定义样式、添加的标注等"人工资产"会被标记并在增量重绘中优先保留，确保人机协作的成果不会因 AI 介入而丢失。
- **支持多轮迭代：** 局部编辑可连续执行，每轮仅触发增量更新。用户可以像"雕刻"一样逐步精修图表，AI 始终沿着用户思路演进。

### 5.3 渐进式反馈机制 (Progressive Feedback)

系统实时可视化展示 5 个 Agent 的工作进度与中间产物。这种"技术透明化"设计不仅有效缓解了用户等待AI生成时的焦虑，更重要的是，它允许用户在关键节点介入，对Agent的决策进行引导或修正，从而实现更深层次的协同演进与控制，确保整个创作过程的透明与可控。（关键设计：实时进度条、Agent决策路径图）
