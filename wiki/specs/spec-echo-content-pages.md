# Spec: 回响内容页（Echo Content Pages）

> **产品向速读**（每页在解决什么问题、本期体验、演进方向）：见 **「回响五页：体验叙事」**。  
> **工程附录**（存储键名、与代码对齐）：见 **「实现真值表」**。  
> **视觉与交互精修**：见 [`spec-echo-visual-polish.md`](./spec-echo-visual-polish.md)。

## 目标
将回响五个模块从侧栏 **占位列表** 演进为：**每一项可进入独立内容页**；页内提供 **结构化信息 + 轻量分析 + 可选 AI 能力**，仍保持 **内向、本地优先、与 Ask 主对话分流**。用户获得的是「与自己有关的深度工作面」，而非第二个文件树或第二聊天窗。

---

## 回响五页：体验叙事（产品）

> 写法对齐常见 **产品说明 / 内部分享**：先讲 **为何而来**，再讲 **这一页此刻交付什么体验**，最后讲 **下一步会加厚哪一层**——不用「不能做什么」当主栏目，那是工程验收表的口气。

| 模块 | 为何而来（用户心智） | 这一页此刻为你做什么 | 下一步会加厚什么 |
|------|---------------------|---------------------|------------------|
| **与你有关** | 少翻目录，也能感到「库里有什么在指向我」。 | 用标题与副文案把「内向、与你相关」讲清楚；「所见」用 **本页专属叙事** 说明隐私与节奏（本地、私密、聚合将接入）；可 **在 MindOS Agent 中继续**，或在配好 AI 后用 **见解** 基于 **当前屏上文字** 写短反思。 | 把文库里的线索 **聚合成列表**（路径、链接、关键词等），在「所见」里逐条可点开。 |
| **未完待续** | 草稿和没收口的回路，想 **一眼续写**。 | 已用 **草稿 / 未收口** 两组版式帮你建立 **心理模型**；同样可进 Agent、用见解。 | 接入真实 **草稿与 TODO**；可加「稍后再提醒」等轻量节奏，不做成第二套任务系统。 |
| **每日回响** | 给今天 **一行空白**，轻到不必为这句话开场长聊。 | **今日一行**：随手写、**离手保存**在本机；**在 MindOS Agent 中继续** 会带上这一行 **预填进对话**。 | 每日 **提示语轮换**、**按日归档** 或写入笔记文件等（产品择版）；详见工程附录中的存储约定演进。 |
| **历史的你** | 在另一个时间刻度上 **瞥见** 旧笔记里的自己。 | 读叙事文案，理解这一格将来会怎么工作；主按钮保留 **占位态**，避免误点空期望。 | **mtime / 抽样算法** 接好以后，「再抽一笔」成为真实随机回访；预览仍坚持 **叙事感**，不做 diff 工具。 |
| **心向生长** | 把 **正在推进的方向** 落成一句话，日后回看 **变与不变**。 | **当前意图**：长句手写、离手保存本机；说明文案标明 **只留在本地**；Agent 与见解可引用你写下的意图（仅在当前页上下文中）。 | **周度自评、极简趋势**；可选 **仅清本模块数据** 的入口；是否同步到 `~/.mindos` 配置另表。 |

### 「所见」与「见解」：两层分工

- **所见**：放 **事实与线索**——将来是列表与卡片；本期用 **按页分叙事的说明卡**（每 segment 一套标题+正文）讲清 **本地、私密、不默认整库上传** 与 **当前进度**，避免空白屏与「假索引」预期。
- **见解**：放 **可选的 AI 短输出**——仅在用户点击后发起；上下文是 **本页拼装的一段摘要**（含模块名、所见文案、以及你在本页已写的「今日一行」「当前意图」等），**不是**默认全文库。使用需在 **设置 → AI** 中配置密钥。

## 实现真值表（工程附录）

> **读者**：实现 / 测试 / 客服排障。与代码一致；分节「数据（P1/P2）」里的设想若与本表冲突，**以本表为准**。产品同事日常以 **「回响五页：体验叙事」** 为准即可。

| 数据 | 存储位置 | 键名 / 约定 | 内容 | 触发写入 |
|------|-----------|-------------|------|-----------|
| 今日一行 | `window.localStorage`（浏览器配置域，随 MindOS Web 所在源） | `mindos-echo-daily-line` | 用户输入的 **整段纯文本**（单槽，非按日多条） | `textarea` **失焦**（`onBlur`）；点「在 Agent 中继续」前会再 `setItem` 一次 |
| 当前意图（心向生长） | 同上 | `mindos-echo-growth-intent` | 意图句纯文本 | `textarea` **失焦** |
| 见解请求 | 无持久化 | — | `POST /api/ask`，body 由 `buildEchoInsightUserPrompt` 拼出 | 用户点 **生成见解** |
| Agent 预填（每日） | 无持久化 | — | `openAskModal(dailyAskPrefill(line))`；`line` 为当前内存中的今日一行（先 persist） | 点 **在 MindOS Agent 中继续**（每日页） |
| Agent 预填（其他 segment） | 无持久化 | — | `openAskModal('回响 / {子模块}\n\n')` | 点 **在 MindOS Agent 中继续** |

**刻意说明（对齐用户疑问）：**

- **今日一行不是「每天一条日记文件」**，也 **不是** `echo-daily-line-YYYY-MM-DD` 按日 key（该形态曾写在早期设想里，**未实现**）。若要「一日一条」版本史，属 **后续产品决策**（可迁每日 key 或写入知识库内 Markdown）。
- **清除方式**：当前无 Echo 专用「清除数据」按钮；开发者可用浏览器清除站点数据，或手动删 `localStorage` 上列键。

## 现状分析
- 当前 `EchoPanel` 仅展示五行标题 + hint +「随后到来」，无路由、无主内容区承载。
- `view/[...path]` 面向单文件编辑；首页面向聚合与 Guide；**缺少专属于 Echo 的 full-width 阅读/反思面**。
- 若在侧栏内嵌长内容，会与 Panel 宽度（~280px）和「可切换面板」模型冲突，且不利于分析块、时间轴、卡片式 AI 输出展示。

## 数据流 / 状态流

```
用户点击 EchoPanel 某行（或访问 /echo）
    → Next.js 导航到 /echo/<segment>（/echo 可重定向到默认 segment）
    → 主内容区渲染 EchoLayout（与现有 Main 同壳：顶栏/面包屑「回响 › 子模块」）
    → 各 segment 页面：
        ├─ 读：文件树 API / mtime / TODO 解析 / ~/.mindos config / 可选 git log
        ├─ 写：意图句、每日一句、snooze、dismiss、偏好（localStorage 或已有 /api/setup 扩展）
        └─ 可选 AI：页内「分析块」显式触发 → 复用 /api/ask 流式或专用短请求；上下文 = 本页可见摘录；不默认持久化到云端

侧栏 EchoPanel（实现演进）：
    - 每行 = Link → /echo/<segment>；pathname 匹配时行样式为选中（amber + 左侧条，与 Rail 一致）
    - 「随后到来」徽章：可在 **整页无 P1 数据** 时保留于侧栏；P1 就绪后改为 **数字角标**（如未完条数）或移除徽章
```

**缓存与一致性：** 内容页以 **客户端拉取 + 短 revalidate** 为主；与首页「最近编辑」共享底层列表时，**同一数据源、不同呈现**（回响页必须带「因何相关」解释层，禁止纯 mtime 复制）。

## Review 纪要（对上一版 spec 的修订）

| 项 | 处理 |
|----|------|
| **路由真值表** | 补充合法 `segment` 枚举；非法 segment → 404 或重定向 `/echo/about-you`。 |

| **移动入口** | Echo 深链在窄屏需可达（Header 菜单或 Drawer 内重复五行链接）；列为 P1 与内容页同步验收。 |

## 全局 UI 壳（所有 Echo 页共用）

### 布局（ASCII）

```
┌─────────────────────────────────────────────────────────────┐
│ App chrome（现有顶栏 / 移动端 Header）                        │
├──────────┬──────────────────────────────────────────────────┤
│ Rail+Panel│  <main> EchoLayout                               │
│ （可选开）│  ┌────────────────────────────────────────────┐   │
│          │  │ 面包屑：回响 › {子模块}     [可选：深入 Ask]   │   │
│          │  ├────────────────────────────────────────────┤   │
│          │  │ A. 页眉区：h1 标题 + 一行副标（muted）        │   │
│          │  ├────────────────────────────────────────────┤   │
│          │  │ B. 事实层（卡片/列表/表单）                   │   │
│          │  ├────────────────────────────────────────────┤   │
│          │  │ C. 分析层 EchoInsight（折叠 panel，可选）    │   │
│          │  ├────────────────────────────────────────────┤   │
│          │  │ D. 行动层：主按钮（打开选中文档等）            │   │
│          │  └────────────────────────────────────────────┘   │
└──────────┴──────────────────────────────────────────────────┘
```

### 设计系统（`21-design-principle`）

- 背景与正文与 `ViewPage` 阅读区一致；强调色 **amber** 仅用于选中、主按钮、链接 hover。
- 卡片：`rounded-lg border border-border bg-card`；间距用既有 `px-4` / `gap-3` 节奏。
- **禁止** Echo 专用新 hex；分析块引用 `muted`、`border`、`text-success`/`text-error` 表示状态。

### 共用组件（建议命名）

| 组件 | 职责 |
|------|------|
| `EchoLayout` | `children` + 面包屑 + optional right action |
| `EchoPageHeader` | `title` / `subtitle` |
| `EchoClueCard` | 路径、标题、相对时间、一行「因何相关」、点击进入 `view` |
| `EchoInsightPanel` | 折叠、上下文摘要列表、「生成见解」、流式/加载/错误/空 |
| `EchoEmptyState` | 插画可选无；一句标题 + 一句说明 + 可选主按钮 |

### 状态（每页）

- **loading**：事实层 skeleton（3 条卡片占位），不闪整块白屏。
- **empty**：`EchoEmptyState`，不调用 AI。
- **error**：一行 `text-error` + 重试；分析块失败 **不**拖垮事实层。

## AI 块统一规范（EchoInsightPanel）

1. **触发前**：展示 bullet 列表「将参考：file1.md 标题、…」最多 5 条；勾选框剔除某条（可选 P2）。**当前实现**为固定说明文案 `insightHint`（不逐条枚举文件，因事实层列表尚未接库）。
2. **按钮文案**：中文「生成见解」/ 英文 "Generate insight"（与 Ask 的「聊天」区分）。
3. **结果**：单条流式 Markdown；**无**多轮历史在 Echo 页内；**当前**无页内「复制」专用按钮（可由用户手动选中文本）。
4. **失败**：`text-error` 摘要 + 「重试」；不抛未捕获错误。
5. **模型**：走用户 Settings 已配 provider；未配置时按钮 **disabled** + 文下 **`generateInsightNoAi`**（非 tooltip）。

## 品味：刻意不做

- 不做 Echo 内第二套全局聊天。
- 不做 streak / 排行榜 / 生产力分数。
- 不默认全文库送模型；默认按需、可勾选、可预览上下文。
- 不把五模块合并为单一无限信息流。

## 与人-AI 分工（摘要）

| 人类 | 系统/AI |
|------|---------|
| 写意图句、每日一句、是否点生成 | 聚合、排序、规则理由 |
| 勾选上下文、关闭 AI | 短见解、失败降级 |
| 清除本模块数据 | 缓存失效 |

## 边界 case 与风险

1. 空库 / 每页独立空态；不强行 AI。  
2. 移动：Echo 入口与深链可达性。  
3. 路径均在 mind root 内；AI 上下文不越权。  
4. 性能：列表 Top N、抽样预算、禁止首屏全量递归。
