# Spec: 回响（Echo）左侧面板

> **演进**：五行模块将各对应独立内容页与分析/可选 AI，见 [`spec-echo-content-pages.md`](./spec-echo-content-pages.md)。

## 目标
在 Activity Bar 提供 **回响 Echo**：专门承载 **与自我相关** 的心智功能（节奏、回望、意图、成长），与 Guide / 首页 / 探索 **明确脱钩**，不在面板内导流至 `/`、`/explore` 或新手任务。

## 现状分析
Guide、Discover、首页各自承担「第一次怎么用」「产品能干什么」「最近写了什么」。回响若复制这些入口，会稀释定位并造成 IA 重复。回响应独占 **心理学向、自我叙事向** 的需求。

## 数据流 / 状态流
- **读**：`useLeftPanel` 的 `activePanel`；各模块未来可接本地索引 / `guideState` 扩展 / 可选 API，**不**作为 Guide 或 Discover 的别名入口。
- **写**：侧栏仍为导航壳；**真实读写**在 `/echo/<segment>` 内容页。产品叙事见该 spec **「回响五页：体验叙事」**；工程键名与请求边界见 **「实现真值表」**。

## 方案（信息架构）

| 模块 | 用户心智 | 实现阶段 |
|------|----------|----------|
| **与你有关** | 知识库里指向「我」的线索 | 占位 → 关联文件、@、最近与你相关的摘要 |
| **未完待续** | 草稿、未闭合的回路 | 占位 → 草稿检测、TODO/Workflow 约定入口 |
| **每日回响** | 今天的轻量反思或提示 | 占位 → 日课模板、可结合日历或本地随机种子 |
| **历史的你** | 时间轴上的旧我 | 占位 → 按日期/版本瞥见旧笔记、git 快照文案 |
| **心向生长** | 目标感与变化 | 占位 → 自选意图句、周期复盘、可选心理向问卷（本地优先） |

- Rail 顺序不变：空间 → **回响** → 搜索 → 插件 → 智能体 → 探索。
- `EchoPanel`：`PanelHeader` + 五行导航（仅标题 + `Link` 至 `/echo/<segment>`；副文案在内容页展示，侧栏不重复）。
- i18n：`panels.echo.*`；英文产品名 Echo。

## Echo **不做**（边界）

- 不放 **Guide** 入口、不链首页「继续引导」。
- 不放 **Discover / 探索** 入口、不列 use case。
- 不复制 **首页**「最近编辑」整块；若未来与首页共享数据，回响侧应是 **意义化摘要**（与你有关），而非同一 UI 的镜像。

## 影响范围
- `EchoPanel.tsx`、`i18n-en.ts`、`i18n-zh.ts`、`i18n-new-keys.test.ts`、本 spec、`wiki/22-page-design.md`（已与回响 IA 一致）。

## 边界 case 与风险
- **期望管理**：侧栏仅标题；内向定位由 `/echo/*` 内容页副文案承担；若用户误找引导，依赖 Rail 其他入口（首页、探索等）。
- **隐私**：意图、历史、每日回响涉及敏感内容，默认本地、可关、可导出说明需在实现模块时写清。
- **与 Ask 边界**：回响可 **引用** 文库生成提示，但不应变成第二对话窗；优先只读摘要或单次卡片。

## 验收标准
- [ ] 回响面板 **无** 指向 `/`、`/explore` 的链接或等价主 CTA。
- [x] 展示五行：与你有关、未完待续、每日回响、历史的你、心向生长（侧栏仅标题；说明在 `/echo/*` 内容页）。
- [x] **侧栏入口**：每行 **链至** `/echo/<segment>`；当前路径匹配时行选中态；有数据后可用角标替代「纯空态」提示（见内容页 spec）。
- [ ] en/zh 键齐全。
- [ ] `npm test`（app vitest）通过。

## 产品设计备忘（品味 × IA）

> 对应 `wiki/21-design-principle.md` 的「内容优先、克制」；回响是 **人的一侧**，且是 **内向面板**（inward），与 Discover 的 **外向**（outward）对称。

### 与 Guide、探索、首页

| 模块 | 问题 | 回响是否替代 |
|------|------|----------------|
| GuideCard | 第一次怎么用？ | 否 |
| Discover | 产品还能干什么？ | 否 |
| 首页 | 最近文件、插件入口 | 否（可共享底层数据，不共享叙事） |
| Echo | 我状态如何？何以为续？ | 是 |

## 愿景锚点：「与人有关」= MindOS 核心定位

> 借用 *defining-product-vision*：**愿景描述用户五年后的世界，而不是功能列表**；借用 *product-taste-intuition*：**味道靠删什么来定义**。

**若 MindOS 在「与人有关」上成功，用户感受到的世界更像：**

- 知识库不是冷文件夹，而是 **持续指向「我」的阅读与写作轨迹**（有归属、可接续）。
- 每一天可以 **用极小成本** 与昨天的自己、上周的意图对一下话，而不必打开 Agent 长聊。
- **敏感与脆弱**（意图、情绪性笔记、未完成）默认留在本地、可解释、可关掉，而不是被产品「推着优化」。

**回响在其中的角色：** 把上述体验 **收束到一个内向面板**（inward），与「教你怎么用」「展示产品能力」彻底分开。Ivan Zhao 式表达：*把激进愿景藏在熟悉的侧栏格子里*——格子像「笔记本边栏」，里层是自我连续性。

**团队决策检验句（vision guides decentralized decisions）：**  
「这个功能是在帮用户感到 **更属于自己**，还是在帮用户 **更会用工件**？」后者归 Discover / Agents / 插件；前者才进 Echo。

## 模块与细节设计（草案）

下列为 **Define → Develop** 之间的规格：每一格有 JTBD、数据与信任、UI 形态、刻意不做、分期。实现时可拆独立子 spec。

### 面板内顺序（当前实现自上而下）

| 顺序 | 模块 | 顺序理由（可迭代） |
|------|------|---------------------|
| 1 | 与你有关 | **与 MindOS「与人有关」定位同构**：先回答「此刻什么在指向我」，再谈日程与历史。 |
| 2 | 未完待续 | 紧接「与我有关」的 **行动面**——未闭合回路，降低从「感知」到「续写」的摩擦。 |
| 3 | 每日回响 | **时间锚「今天」**；放在中段，避免顶栏像日历 App、底栏像档案馆。 |
| 4 | 历史的你 | **更长尺度回望**；与「今天」形成纵向对照。 |
| 5 | 心向生长 | **元层**：周/月尺度与心理向；放底部符合「由近及远、由事及心」。 |

> *假设*：若数据证明用户更先需要「今日一句」，可将「每日回响」与「与你有关」互换；顺序应在有使用后用定性+点击热力再定。

### 打开回响时的默认焦点（首屏）

| 决策 | 建议 | 说明 |
|------|------|------|
| **默认落在哪一节** | **「与你有关」**（列表第一条 / 手风琴默认展开第一节） | 与 MindOS「与人有关」核心定位一致；用户打开 Echo 的第一眼应是「什么在指向我」，而不是日历感或待办感。 |
| **空态** | 仍在首屏展示「与你有关」的空态文案 | 不自动滚到下面某节，避免「上面是空的像坏掉」。 |
| **将来有折叠 UI 时** | 默认只展开第一节；可选 `localStorage` 记忆「上次展开」 | 老手若总点开「每日回响」，可记住偏好，但不作为首次安装的默认。 |
| **若战略改押「日课」** | 才把默认展开改为「每日回响」 | 属于定位取舍：**人-线索优先** vs **今日仪式优先**；二选一写进 release note。 |

### 排序备选（与当前实现对照）

| 方案 | 顺序 | 适合叙事 | 代价 |
|------|------|----------|------|
| **A 叙事型（当前）** | 与你有关 → 未完 → 每日 → 历史 → 心向 | 先「被谁碰到」→ 再「没收尾」→ 再「今天」→ 再「往日」→ 最后「元层」 | 急着「先写今天一句」的人要多滑一块 |
| **B 时间钉顶** | 每日回响 → 与你有关 → 未完 → 历史 → 心向 | 强「今日仪式」、日记用户 | 削弱「与你有关」作为第一视觉锚点 |
| **C 行动钉顶** | 未完待续 → 与你有关 → 每日 → 历史 → 心向 | GTD、草稿地狱、先清 loop | 「与人有关」让位给 urgency，品牌叙事弱一档 |

**产品建议**：维持 **方案 A**；若上线后数据或访谈显示「打开 Echo 多半为日课」，再评估改为 B 或仅调整默认展开节（不必改全局排序）。

### 1. 与你有关

| 维度 | 规格 |
|------|------|
| **JTBD** | 「在不翻遍树与搜索的前提下，**瞥见此刻与『我』强相关的几条线索**。」 |
| **可能数据源（本地优先）** | 最近编辑但 **标题/路径/ frontmatter 含自我锚点**（如 `我`/`journal`/`日记`/用户配置的关键词）；双向链接指向「关于我」类笔记；可选：@ 提及当前用户 profile 名的文件（若有）。 |
| **UI 形态** | 侧栏内 **3～5 条**，每条：短标题 + 相对时间 + 可选一行「为何与你有关」的生成/规则说明（极短）；点击进入 `view`。空态：一句安静说明 + 不推销功能。 |
| **交互细节** | 不提供批量操作；**无 infinite scroll**（避免侧栏变第二文件树）。刷新策略：打开面板或聚焦窗口时增量拉取，可配「免打扰」时段。 |
| **刻意不做** | 不做完整「最近编辑」镜像；不做社交/协作「与你有关」；首版不上需要上传外泄的模型推断。 |
| **分期** | **P1**：规则/关键词 + 最近 mtime Top N 过滤；**P2**：轻量链接图或 frontmatter 约定。 |

---

### 2. 未完待续

| 维度 | 规格 |
|------|------|
| **JTBD** | 「我**明确还没合上**的东西在哪：草稿、勾了一半的待办、写到一半的笔记。」 |
| **可能数据源** | `Untitled`/draft 路径约定；`TODO.md` / workflow 注册表中的 open 项（若已有解析）；最近打开但未保存缓冲（仅桌面端需谨慎）；Git 未提交文件列表（可选，噪声大需过滤）。 |
| **UI 形态** | **分组小块**：草稿 / 待办 / 其他；每组 **≤3 条** 展开，多出的「还有 N 项」链到专用视图或搜索（链到搜索不算 Discover，算工具延伸，可后续争议再定）。每条主操作：**打开续写**。 |
| **交互细节** | 支持 **Dismiss snooze**（「明天再提醒」）需本地存储键，避免骚扰。 |
| **刻意不做** | 不做全项目 Issue 管理；不与 Guide 任务混排。 |
| **分期** | **P1**：仅草稿文件名 + TODO 解析；**P2**：工作流约定、snooze。 |

---

### 3. 每日回响

| 维度 | 规格 |
|------|------|
| **JTBD** | 「**今天**用一句话或一个小问题，和文库或和昨天的自己 **轻量对齐**，不必开聊天。」 |
| **可能数据源** | 本地模板库（Markdown 片段轮播）；可选：昨日编辑过的 1 个文件标题作锚点（无模型也可）；**可选** LLM 生成每日一问（默认关、显式开启、本地 key）。 |
| **UI 形态** | **单卡**：日期 + 一句提示/问题 + 「换一句」（同种子或下一模板）；可选单行输入「今日一句」仅存本地。整卡高度有上限，不滚动成文章。 |
| **交互细节** | 与 Ask **互斥展示**：从回响「深入聊」可 **单次** 带上下文打开 Ask，但默认不回声面板内嵌对话流。 |
| **刻意不做** | 不做每日推送通知（首版）；不做「打卡 streak」绑架（*taste*：警惕游戏化侵蚀安静感）。 |
| **分期** | **P1**：纯模板轮播 + 本地一句；**P2**：锚点文件 + 可选模型。 |

---

### 4. 历史的你

| 维度 | 规格 |
|------|------|
| **JTBD** | 「**看见过去某个时间点的自己**在笔记里留下了什么——不是 diff 工具，是 **叙事性一瞥**。」 |
| **可能数据源** | 按日随机/伪随机抽一篇 **去年同日附近** 修改的笔记（本地 mtime）；Git `git log` 某路径一句话摘要（高级、可选）；仅展示路径与首段预览，默认不自动展开全文。 |
| **UI 形态** | **一条主卡** + 「再看一条」；预览 **2～3 行** prose，模糊敏感词可选（后期）。 |
| **交互细节** | 明确标注「来自 yyyy-mm-dd 附近」，避免用户误以为「实时心态」。 |
| **刻意不做** | 不做完整时间轴浏览器（那是独立页）；不做「去年今日」社交分享样式。 |
| **分期** | **P2**：mtime 抽样；**P3**：Git 历史一句、多版本切换。 |

---

### 5. 心向生长

| 维度 | 规格 |
|------|------|
| **JTBD** | 「我**这周/这月在在乎什么**，以及 **变了吗**——偏心理与元认知，非 KPI 仪表盘。」 |
| **可能数据源** | 用户 **手写** 意图句（`~/.mindos` 或 config）；可选周期性「同一问」自评滑条（1–5）仅存本地；**禁止**默认上传。 |
| **UI 形态** | 顶部 **当前意图一句**（可编辑）；下方 **极简时间折线或小点**（近 4 周自评，可无则空）；链接到「导出/清除我的回响数据」。 |
| **交互细节** | 编辑即保存 debounce；清空需二次确认。 |
| **刻意不做** | 不做「生产力分数」排行；不与 Agent 默认共享意图句（除非用户显式 @）。 |
| **分期** | **P1**：单句意图 + localStorage；**P2**：周回顾模板 + 自评点；**P3**：可选加密备份。 |

## 横切：信任、性能、无障碍

- **隐私默认值**：Echo 产生的 UGC（意图句、每日一句）**默认不出库**；任何云同步须在设置中单开，并在本 spec 数据流节补审计说明。
- **性能**：侧栏打开 **首屏 <100ms 感知**（占位或缓存）；重扫描后台异步，避免卡 Rail。
- **a11y**：每模块一块 `region` + `aria-labelledby`；「即将推出」在实现真实内容后改为 live region 适度播报（可选）。

## 验收标准（增补 · 面向实现阶段）

当某模块从占位进入 **P1 实现** 时，需满足：

- [ ] 该模块的 **JTBD** 与上表一致或可文档化变更理由。
- [ ] 有 **刻意不做** 清单的对照检查（防范围 creep）。
- [ ] 数据读写路径在 **数据流 / 状态流** 节有单独子段落；敏感数据有清除路径。
- [ ] UI 符合 `21-design-principle`（无违规 hex、focus-visible、侧栏信息密度上限）。
