# 产品建议书 (Product Proposal): MindOS

## 愿景 (Vision)

**“人类在此思考，Agent 依此行动。”**

MindOS 不仅仅是一个本地知识库，它是一个 **“人机协同心智平台” (Human-AI Collaborative Mind Platform)**。它提供了一个极致简洁、反应迅速的浏览器界面，让用户能够以前所未有的方式管理、编辑并与自己的“第二大脑”对话。更重要的是，它通过标准化的 MCP 协议，将这些人类思维碎片转化为 AI Agent 可以理解并执行的结构化指令，实现真正的人机共生。

## 痛点分析 (Problem)

- **心智隔阂：** 人类的笔记（Notion/Obsidian）是给人的，Agent 的记忆（向量数据库）是给机器的。两者互不通气，导致 Agent 难以理解人类的深度背景。
- **Agent 黑箱：** 现有的 AI 助手运行过程不可见，用户无法直接查看、干预或纠正 Agent 的中间思考过程和沉淀的知识。
- **管理负担：** 传统的 Markdown 文件夹管理繁琐，MindOS 通过“解放文档管理”理念，实现人机上下文的高效自动化同步。
- **编辑断层：** 在查看笔记与修改笔记之间往往需要切换编辑器，缺乏“即看即改”的丝滑感。
- **孤岛效应：** 文件之间缺乏动态连接，难以在对话中快速引用多个相关上下文。

## 目标用户 (Target User)

- **AI 独立开发者/创始人：** 需要管理复杂的 SOP、产品路线图和技术架构。
- **系统性思考者：** 拥有庞大的本地 Markdown/CSV 知识库，追求极致的本地控制与隐私。
- **Agentic AI 深度用户：** 希望利用 MCP 协议让 AI Agent 能够精准读写自己个人上下文的用户。

## 核心功能 (Features)

### 已实现 (Shipped)
- **人机协同 UI：** 琥珀色 (Warm Amber) 设计系统，专为长文阅读和沉淀思考优化的排版。支持亮/暗主题切换。
- **双模式编辑器：** TipTap 富文本 (WYSIWYG) + CodeMirror 6 Markdown 源码编辑，双模式无缝切换。
- **人机共享思维 (Shared Mind)：**
  - **MCP Server (20+ 工具)：** 读写、搜索、语义编辑、行级操作、反向链接、Git 历史、Bootstrap 上下文加载。
  - **双传输模式：** stdio（本地 Agent）+ Streamable HTTP（远程设备），Bearer Token 认证。
  - **API 驱动：** 10 个 REST API 端点（文件 CRUD、搜索、反向链接、图谱、AI 对话、PDF 提取、设置管理）。
- **MindOS Skills：** Agent 工作流技能（EN + ZH），定义结构感知路由、搜索回退策略、多文件审批流程等最佳实践。
- **AI 对话助手 (⌘/)：** Vercel AI SDK 流式输出，自动注入 Bootstrap + Skill 上下文，支持 `@` 引用本地文件 + 文件上传（PDF 提取），多轮对话。
- **全局搜索 (⌘K)：** 毫秒级全文搜索，带 snippet 预览。
- **CSV 快速追加：** 无需进入编辑模式，直接在表格视图添加新行（适合记录日常数据）。
- **侧边栏同步：** 导航时自动在文件树中定位当前文件。
- **关联视图 (Backlinks)：** 自动索引全库，展示引用了当前文件的相关上下文。
- **可视化引用图谱 (Wiki Graph)：** 动态解析文件间 WikiLink + Markdown Link 引用关系，支持 Local/Global 两种范围。
- **多端支持：** 响应式移动端布局，适配碎片化阅读场景。
- **安全机制：** Bearer Token 认证（保护 API + MCP HTTP）、路径沙箱、INSTRUCTION.md 写保护、原子写入。
- **结构化模板：** 中英双语预设模板（Profile, Workflows, Connections, Resources 等）。
- **CI/CD：** GitHub Actions 自动同步 `mindos-dev` → `mindos`，Landing Page 自动部署 gh-pages。
- **11+ Agent 兼容：** MindOS Agent, Claude Code, Cursor, Windsurf, Cline, Gemini CLI, GitHub Copilot 等。

### 优先级 P1 (Next)
- **ACP (Agent Communication Protocol)：** 连接外部 Agent（如 Claude Code、Cursor），让知识库成为多 Agent 协作中枢。
- **深度 RAG 集成：** 基于知识库内容的检索增强生成，让 AI 回答更精准、更有上下文。
- **反向链接视图 (Backlinks View)：** 展示所有引用当前文件的反向链接，理解笔记在知识网络中的位置。
- **Agent 审计面板 (Agent Inspector)：** 将 Agent 操作日志渲染为可筛选的时间线，审查每次工具调用的详情。
- **评论/批注机制：** Agent 在文件旁添加批注而非直接修改，支持人机异步协作。

### 优先级 P2 (Future / Visionary)
- **动态技能协议 (Dynamic Skill Manifest)：** 允许在 Markdown 中直接定义自动化脚本（如 `:::skill`），笔记即工具，知识库即 Agent 的能力说明书。
- **心智时光机 (Mind State Time-lapse)：** 溯源知识形成过程，可视化展示 SOP 如何在 Agent 反思与人类纠偏中逐渐成熟。
- **跨 Agent 协同网格 (Agent-to-Agent Mesh)：** 定义冲突解决协议，让不同厂商的 Agent 围绕同一个 MindOS 协作，实现“一个大脑，多个分身”。

## 交互原则 (UX Principles)

- **极速响应 (Speed First)：** 拒绝 Loading 焦虑，内容即开即读。
- **极简主义 (Minimal Chrome)：** 界面只保留内容与搜索，让用户专注于思考本身。
- **键盘驱动 (Keyboard-driven)：** 所有的核心动作（搜索、编辑、对话、设置）均可通过快捷键完成。
- **本地掌控 (Localism)：** 彻底的本地隐私，数据永远属于用户。
