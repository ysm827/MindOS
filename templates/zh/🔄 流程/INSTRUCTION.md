# 流程目录 Instruction Set

## 目标

- 定义 `🔄 流程/` 目录的局部执行规则。
- 保证 SOP 文档可执行、可复用、可维护。

## 局部规则

- 先读取根 `INSTRUCTION.md`，再读取本文件。
- 流程类文档必须步骤化，避免仅概念描述。
- 每个子目录应聚焦单一场景（如科研、媒体、信息、配置）。
- 涉及命令或操作序列时，优先给出可直接执行的最小步骤。

## 执行顺序

1. 根 `INSTRUCTION.md`
2. 本目录 `INSTRUCTION.md`
3. 本目录 `README.md` 与目标文件

## 边界

- 与根规则冲突时，根规则优先。

## 同步规则（流程）

- 新增场景子目录时：同步更新 `🔄 流程/README.md` 的目录结构。
- 重命名或移动 SOP 时：同步更新所有引用路径。
- 流程被替换或废弃时：在相关文档标注状态，避免执行旧流程。
