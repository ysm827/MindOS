# 画像目录 Instruction Set

## 目标

- 定义 `👤 画像/` 目录的局部执行规则。
- 保证用户画像信息稳定、可追踪、可复用。

## 局部规则

- 先读取根 `INSTRUCTION.md`，再读取本文件。
- 画像类文件用于长期稳定信息，不放临时草稿。
- 偏好、身份、目标等核心信息变更时，应保持跨文件一致。
- 涉及用户关键偏好时，优先更新对应源文件而不是重复写在多处。

## 执行顺序

1. 根 `INSTRUCTION.md`
2. 本目录 `INSTRUCTION.md`
3. 本目录 `README.md` 与目标文件

## 边界

- 与根规则冲突时，根规则优先。

## 同步规则（画像）

- 新增画像字段或文档时：同步更新 `👤 画像/README.md` 结构说明。
- 修改已有偏好或目标时：同步检查相关文件，避免语义冲突。
